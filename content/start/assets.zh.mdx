**什么是资源？** 资源是指所有非代码文件，例如图像、声音和视频。

你可以使用保存在项目本地的资源或在线资源（对于第二种方式，你需要确保所使用的云服务允许 *CORS 请求*）。 如果你的资源托管在线上，你的游戏将需要互联网连接。 如果没有网络连接，你应该通知用户并阻止游戏继续运行。

如果你正在创建视觉小说，建议将频繁使用的资源保存在本地。 对于游戏中仅使用一次的资源，最好将其发布到线上。

要加载和操作资源（图像、gif、视频等）， 需要使用 `Assets`。 `Assets` 是一个功能丰富的类，来自 PixiJS 库。 更多信息请参阅[此处](https://pixijs.com/8.x/guides/components/assets)。 阅读本文档以更好地<DynamicLink href="/start/assets-management">组织资源的上传</DynamicLink>也非常重要。

你主要有两种保存资源的选择：本地或线上。

## 本地资源

要在本地保存和使用资源，你可以使用任何文件夹——没有限制。 但是，建议使用 `assets` 文件夹。 在此文件夹内，你可以创建子文件夹以更好地组织资源。

以下是如何导入资源并将其加载到项目中的示例：

```ts title="utils/assets.ts"
import { Assets } from "@drincs/pixi-vn";
import bg01hallway from "../assets/images/bg01-hallway.webp";

Assets.add({
    alias: "bg01-hallway",
    src: bg01hallway,
});
```

## 资源托管

你可以将资源保存在线上。 如果你想节省计算机空间，这是一个不错的选择。 你可以使用任何允许上传文件并生成公共 URL（已启用 CORS）的云服务。

以下是如何导入资源并将其加载到项目中的示例：

```ts title="utils/assets.ts"
import { Assets } from "@drincs/pixi-vn";

Assets.add({
    alias: "bg01-hallway",
    src: "https://raw.githubusercontent.com/DRincs-Productions/pixi-vn-bucket/refs/heads/main/breakdown/bg01-hallway.webp",
});
```

你可以完全自由地按自己的意愿保存资源。 如果你计划将资源保存在线上，以下是一些可选方案：

<Accordions>

<Accordion title="GitHub" id="GitHub">

你可以使用 GitHub 托管资源。 你可以在项目中使用文件的原始链接（raw link）。 链接将采用以下格式：

```text
https://raw.githubusercontent.com/[repository]/raw/refs/heads/main/[file path]
```

* **价格**：完全免费。
* **空间限制**：无空间限制，但每个单独文件不得超过 100 MB。
* **文件类型**：你可以上传任何类型的文件。
* **速度**：速度并非最佳。
* **编辑资源**：你可以在保持相同 URL 的同时编辑文件。

</Accordion>

<Accordion title="图片托管" id="image-hosting">

图片托管是一种允许你上传图片的服务。 有很多免费上传图片的网站，例如 [imgbb](https://imgbb.com/)、[imgix](https://www.imgix.com/)、[imgur](https://imgur.com/)。 你可以在项目中使用图片链接。

* **价格**：完全免费，但你可以付费获取更多功能。
* **空间限制**：无空间限制，但每个单独文件可能有最大大小限制。
* **文件类型**：只能上传图片。
* **速度**：速度良好。
* **编辑资源**：无法在保持相同 URL 的同时编辑文件。

</Accordion>

<Accordion title="云存储" id="Cloud-storage">

云存储是一种允许你将文件上传到线上的服务。

* **价格**：通常为付费服务，或提供有限制的免费版本。
* **空间限制**：按月根据使用空间计费（通常不超过特定阈值时免费）。
* **文件类型**：你可以上传任何类型的文件。
* **速度**：速度良好。
* **编辑资源**：你可以在保持相同 URL 的同时编辑文件。

以下是一些最受欢迎的云存储服务列表：

* ![firebase-text](/firebase.svg) **Firebase Storage** 是一个非常易于使用的云服务。 Firebase 有两个套餐：Spark（免费）和 Blaze（按量付费）。 你可以[在此处](https://firebase.google.com/pricing)找到更多信息。\
  **解决 Firebase Storage 的 CORS 问题**：
  * 安装 [gcloud CLI](https://cloud.google.com/sdk/docs/install)
  * 阅读此[文档](https://medium.com/@we.viavek/setting-cors-in-firebase-19a2cce2fe28)以解决 CORS 问题。
* ![aws-text](/aws.svg) **Amazon S3** 是一种云服务。 与竞品相比，它提供了丰富的设置选项，但可能更难上手。 使用 Amazon S3 需要付费套餐。 你可以[在此处](https://aws.amazon.com/s3/pricing/)找到更多信息。
* ![supabase-text](/supabase.svg) **Supabase** 是一个开源的 Firebase 替代方案。 Supabase 有两个套餐：免费和按量付费。 你可以[在此处](https://supabase.io/pricing)找到更多信息。
* **Convex** 是一种云服务，允许你存储和提供用户生成的内容，例如照片、视频或其他文件。 Convex 有两个套餐：免费和按量付费。 你可以[在此处](https://www.convex.dev/pricing)找到更多信息。

## 其他功能

<Accordions>

<Accordion title="缓存资源" id="caching-assets">

<Callout title="模板" type="info">

所有模板中都包含一个 Service Worker，用于缓存游戏中使用的 外部资源（图像、声音、视频等）。 这可以提高性能并减少玩家的加载时间。

</Callout>

对于托管在线的资源，Service Worker 会在首次下载后将其缓存到用户设备本地。 这意味着对同一资源的后续请求将从缓存中提供，从而实现更快的加载速度并减少数据使用量。

```js tab="public/service-worker.js"
const CACHE_NAME = "external-assets-v1";
const MAX_AGE_DAYS = 7;
const MAX_AGE_MS = MAX_AGE_DAYS * 24 * 60 * 60 * 1000;

const lastUsed = {};

// The fetch event handler below implements a simple cache-for-host strategy:
// - It only attempts to cache responses for requests whose hostname is explicitly allowed in the switch below.
// - Cached entries are kept in the "external-assets-v1" cache and tracked via `lastUsed` timestamps.
// - If a cached response exists and is not older than MAX_AGE_DAYS, it is served directly.
// - Otherwise the service worker fetches from the network, caches the successful response, and returns it.
// - If the network fails and a cached response exists, the cached response is returned as a fallback.
// NOTE: Add the hostnames of external asset providers you want to cache (CDNs, raw asset hosts, etc.) in the switch below.
self.addEventListener("fetch", (event) => {
    const request = event.request;
    const url = new URL(request.url);

    switch (url.hostname) {
        case "raw.githubusercontent.com":
            break;
        // Add asset hostnames here to enable caching for them.
        // Examples:
        // case "raw.githubusercontent.com":
        // case "cdn.jsdelivr.net":
        // case "your.cdn.domain.com":
        //     break;
        default:
            return;
    }

    event.respondWith(
        (async () => {
            const cache = await caches.open(CACHE_NAME);
            const now = Date.now();
            const cached = await cache.match(request);

            if (cached && lastUsed[url.href]) {
                const age = now - lastUsed[url.href];
                if (age < MAX_AGE_MS) {
                    lastUsed[url.href] = now;
                    return cached;
                }
                await cache.delete(request);
                delete lastUsed[url.href];
            }

            try {
                const response = await fetch(request);
                if (response && response.type === "cors" && response.status < 400) {
                    await cache.put(request, response.clone());
                    lastUsed[url.href] = now;
                }
                return response;
            } catch (err) {
                if (cached) return cached;
                return fetch(request);
            }
        })()
    );
});
```

```ts tab="main.ts"
// Register service worker
if ("serviceWorker" in navigator) {
    window.addEventListener("load", () => {
        navigator.serviceWorker.register("/service-worker.js").catch(console.error);
    });
}
```
