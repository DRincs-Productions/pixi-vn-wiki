---
title: Erzählfluss mit Labels verwalten
description: Beschreibt, wie der Erzählfluss in Pixi'VN mithilfe von Labels gesteuert wird, einschließlich der Funktionen call, jump und Labels schließen.
---

Der Erzählfluss wird durch Funktionen wie `call`, `jump`, `continue`, `back` und `closeLabel` gesteuert, die alle im `narration`-Objekt verfügbar sind.

## Ein Label ausführen

:::tip[*ink*]

Sie können diese Methode mit der *ink*-Syntax verwenden. Mehr dazu <DynamicLink href="/ink/labels#run-a-knot">hier</DynamicLink>.

:::

<Accordions>

<Accordion title="Ein `label` `call`en" id="call-a-label">

Um ein `label` aufzurufen, verwende die Funktion [`narration.call`](/jsdoc/pixi-vn/index/interfaces/NarrationManagerInterface#call).

```ts
import { narration } from "@drincs/pixi-vn";

narration.call(startLabel, {});
```

Wenn du ein `label` aufrufst, wird der erste `step` dieses `label` ausgeführt. Wenn zuvor ein anderes `label` lief, werden die verbleibenden `steps` dieses vorherigen `label` fortgesetzt, nachdem das aufgerufene `label` beendet ist.

Wenn zum Beispiel das Spiel `step` 5 von `label` A ausführt und du `label` B **aufrufst**, setzt das Spiel nach der Ausführung aller `steps` von `label` B mit `step` 6 von `label` A fort.

[`narration.call`](/jsdoc/pixi-vn/index/interfaces/NarrationManagerInterface#call) gibt das <DynamicLink href="/start/labels#step-results">Ergebnis</DynamicLink> des ersten `step` des aufgerufenen `label` zurück.

---

Wenn du [`narration.call`](/jsdoc/pixi-vn/index/interfaces/NarrationManagerInterface#call) innerhalb eines `step` ausführst, solltest du das Ergebnis von [`narration.call`](/jsdoc/pixi-vn/index/interfaces/NarrationManagerInterface#call) zurückgeben und `await` verwenden. Dies stellt sicher, dass der Verlauf korrekt verwaltet wird.

```ts title="content/labels/start.label.ts"
import { narration, newLabel } from "@drincs/pixi-vn";

export const startLabel = newLabel("start", [
    async (props) => {
        return await narration.call(TestLabel, props);
    },
    async (props) => await narration.call(TestLabel, props),
]);
```

</Accordion>

<Accordion title="Zu einem `label` `jump`en" id="jump-to-a-label">

Um zu einem `label` zu springen, verwende die Funktion [`narration.jump`](/jsdoc/pixi-vn/index/interfaces/NarrationManagerInterface#jump).

```ts
import { narration } from "@drincs/pixi-vn";

narration.jump(startLabel, {});
```

Wenn du zu einem `label` springst, werden die `steps` des aktuellen `label` gestoppt und die `steps` des neuen `label` beginnen.

Wenn zum Beispiel das Spiel `step` 5 von `label` A ausführt und du `label` B **aufrufst**, aber dann zu `label` C **springst**, setzt das Spiel nach dem Ende von `label` C mit `step` 6 von `label` A fort. Wenn du zu `label` C springst, wird `label` B geschlossen.

[`narration.jump`](/jsdoc/pixi-vn/index/interfaces/NarrationManagerInterface#jump) gibt das <DynamicLink href="/start/labels#all-steps-result">Ergebnis des ersten `step` des aufgerufenen `label`</DynamicLink> zurück.

---

Wenn du [`narration.jump`](/jsdoc/pixi-vn/index/interfaces/NarrationManagerInterface#jump) innerhalb eines `step` ausführst, solltest du das Ergebnis von [`narration.jump`](/jsdoc/pixi-vn/index/interfaces/NarrationManagerInterface#jump) zurückgeben und `await` verwenden. Dies stellt sicher, dass der erste `step` des neuen `label` abgewartet wird.

```ts title="content/labels/start.label.ts"
import { narration } from "@drincs/pixi-vn";

export const startLabel = newLabel("start", [
    async (props) => {
        return await narration.jump(TestLabel, props);
    },
    async (props) => await narration.jump(TestLabel, props),
]);
```

</Accordion>

<Accordion title="Das Spiel starten" id="start-game">

Um das Spiel zu starten, verwende die Funktion [`Game.start`](/jsdoc/pixi-vn/index/namespaces/Game/functions/start). Diese Funktion entspricht dem Aufrufen eines `label`, es wird jedoch empfohlen, sie für einen saubereren Start zu verwenden.

```ts title="content/labels/start.label.ts"
import { Game } from "@drincs/pixi-vn";

Game.start(startLabel, gameProps);
```

</Accordion>

</Accordions>

## Steps fortsetzen und zurückgehen

### Fortsetzen

:::tip[UI-Bildschirm]

Ein Beispiel für den "Weiter"-Button findet sich im Abschnitt <DynamicLink href="/start/interface-examples#go-next">Interface-Beispiele</DynamicLink>.

:::

Um den nächsten `step` auszuführen, verwende die Funktion [`narration.continue`](/jsdoc/pixi-vn/index/interfaces/NarrationManagerInterface#continue).

```ts
import { narration } from "@drincs/pixi-vn";

await narration.continue({});
```

[`narration.continue`](/jsdoc/pixi-vn/index/interfaces/NarrationManagerInterface#continue) ist asynchron, daher kann `.then` verwendet werden, um zum Beispiel einen "Weiter"-Button zu deaktivieren, bis der `step` abgeschlossen ist.

```ts
import { narration } from "@drincs/pixi-vn";

// disable next button
narration.continue({}).then((result) => {
    // enable next button
});
```

<Accordions>

<Accordion title="continue innerhalb eines `step` ausführen" id="call-continue-inside-a-step">

Wenn du [`narration.continue`](/jsdoc/pixi-vn/index/interfaces/NarrationManagerInterface#continue) innerhalb eines `step` aufrufst, wird die "continue"-Anfrage in die Warteschlange eingereiht und ausgeführt, wenn die Warteschlange leer ist.

```ts title="content/labels/start.label.ts"
import { narration, newLabel } from "@drincs/pixi-vn";

export const startLabel = newLabel("start", [
    async (props) => {
        narration.continue(props);
    },
]);
```

Hier ist ein Beispiel, das die Funktionsweise der Warteschlange veranschaulicht:

```ts title="content/labels/start.label.ts"
import { narration, newLabel } from "@drincs/pixi-vn";

export const startLabel = newLabel("start", [
    async (props) => {
        await narration.call(label2, props);
        console.log(1);
    },
    () => {
        console.log(3);
    },
]);

const label2 = newLabel("label_02", [
    async (props) => {
        await narration.continue(props);
        console.log(2);
    },
]);
```

In diesem Beispiel lautet die Ausgabe `2`, `1`, `3`. Die Schritte sind:

1. `await narration.call(label2, props)` ruft label2 auf und wartet auf dessen ersten `step`. (1 Element in der Warteschlange)
2. Der erste `step` von label2 führt `await narration.continue(props)` aus, aber die continue-Anfrage wird in die Warteschlange eingereiht, weil die Warteschlange nicht leer ist. (2 Elemente in der Warteschlange)
3. `console.log(2)` wird ausgeführt und beendet den ersten `step` von label2. (1 Element in der Warteschlange)
4. `console.log(1)` wird ausgeführt und beendet den ersten `step` von startLabel. (0 Elemente in der Warteschlange)
5. Die continue-Anfrage wird jetzt ausgeführt, und der zweite `step` von startLabel läuft. (1 Element in der Warteschlange)
6. `console.log(3)` wird ausgeführt und beendet den zweiten `step` von startLabel. (0 Elemente in der Warteschlange)

</Accordion>

<Accordion title="Prüfen, ob zum nächsten `step` gewechselt werden kann" id="can-go-next">

Die Eigenschaft [`narration.canContinue`](/jsdoc/pixi-vn/index/interfaces/NarrationManagerInterface#cancontinue) kann verwendet werden, um zu prüfen, ob der nächste `step` ausgeführt werden kann.

[`narration.canContinue`](/jsdoc/pixi-vn/index/interfaces/NarrationManagerInterface#cancontinue) ist `false`, wenn:

* Ein `step` ausgeführt wird
* Der Spieler eine <DynamicLink href="/start/choices">Entscheidung treffen</DynamicLink> muss
* Der Spieler einen <DynamicLink href="/start/input">Wert eingeben</DynamicLink> muss

```tsx title="components/NextButton.tsx"
import { narration } from "@drincs/pixi-vn";

function NextButton() {
    return (
        <button
            disabled={!narration.canContinue}
            onClick={() => {
                narration.continue({});
            }}
        >
            Next
        </button>
    );
}
```

</Accordion>

</Accordions>

### Zurückgehen

:::tip[UI-Bildschirm]

Ein Beispiel für den Zurück-Button findet sich im Abschnitt <DynamicLink href="/start/interface-examples#go-back">Interface-Beispiele</DynamicLink>.

:::

Bei jedem `step` speichert das System den aktuellen Zustand des Spiels. Um zum vorherigen `step` zurückzugehen, verwende die Funktion [`stepHistory.back`](/jsdoc/pixi-vn/index/interfaces/HistoryManagerInterface#back).

<Accordions>

<Accordion title="Prüfen, ob zurückgegangen werden kann" id="can-go-back">

Die Eigenschaft [`stepHistory.canGoBack`](/jsdoc/pixi-vn/index/interfaces/HistoryManagerInterface#cangoback) kann verwendet werden, um zu prüfen, ob ein Zurückgehen möglich ist.

[`stepHistory.canGoBack`](/jsdoc/pixi-vn/index/interfaces/HistoryManagerInterface#cangoback) ist `false`, wenn keine `steps` im Verlauf zum Wiederherstellen vorhanden sind.

```ts
import { stepHistory } from "@drincs/pixi-vn";

if (stepHistory.canGoBack) {
    stepHistory.back();
}
```

</Accordion>

<Accordion title="Die Möglichkeit des Zurückgehens blockieren" id="block-the-possibility-of-going-back">

Das "Zurückgehen" kann durch den Aufruf von [`stepHistory.blockGoBack`](/jsdoc/pixi-vn/index/interfaces/HistoryManagerInterface#blockgoback) blockiert werden.

```typescript
import { stepHistory } from "@drincs/pixi-vn";

stepHistory.blockGoBack();
```

</Accordion>

</Accordions>

## Labels schließen

<Accordions>

<Accordion title="Aktuelles `label` schließen" id="close-current-label">

Um das aktuelle `label` zu schließen, verwende die Funktion [`narration.closeCurrentLabel`](/jsdoc/pixi-vn/index/interfaces/NarrationManagerInterface#closecurrentlabel).

```typescript
import { narration } from "@drincs/pixi-vn";

narration.closeCurrentLabel();
```

</Accordion>

<Accordion title="Alle `labels` schließen" id="close-all-labels">

Um alle `labels` zu schließen, verwende die Funktion [`narration.closeAllLabels`](/jsdoc/pixi-vn/index/interfaces/NarrationManagerInterface#closealllabels). **Wenn du diese Funktion aufrufst und danach kein `label` aufrufst, endet das Spiel**

```typescript
import { narration } from "@drincs/pixi-vn";

narration.closeAllLabels();
```

</Accordion>

</Accordions>

<Accordion title="Rollback und Rollforward" id="rollback-rollforward">

:::tip[Vorlagen]

In allen Templates ist diese Implementierung bereits enthalten.

:::

Wenn du mehrere [`stepHistory.back`](/jsdoc/pixi-vn/index/interfaces/HistoryManagerInterface#back)- und [`narration.continue`](/jsdoc/pixi-vn/index/interfaces/NarrationManagerInterface#continue)-Anfragen synchron ausführst, reiht das System sie intern in eine Warteschlange ein und berechnet das Delta, um nur die notwendigen Prozesse auszuführen.

Du kannst daher Rollback- und Rollforward-Funktionen mithilfe der Funktionen [`stepHistory.back`](/jsdoc/pixi-vn/index/interfaces/HistoryManagerInterface#back) und [`narration.continue`](/jsdoc/pixi-vn/index/interfaces/NarrationManagerInterface#continue) implementieren.

```tsx title="lib/hooks/quick-tools-hooks.ts"
import { HTML_CANVAS_LAYER_NAME, HTML_UI_LAYER_NAME } from "@/constans";
import { useGameProps } from "@/lib/hooks/props-hooks";
import { GameStatus } from "@/lib/stores/game-status-store";
import { hasScrollableParent } from "@/lib/utils/scroll-utils";
import { type StepLabelProps, stepHistory } from "@drincs/pixi-vn";
import { narration } from "@drincs/pixi-vn/narration";
import { useThrottler } from "@tanstack/react-pacer";
import { useEffect, useRef } from "react";

function isInsideRoot(target: EventTarget | null, selector: string): boolean {
    if (!(target instanceof HTMLElement)) return false;
    return target.closest(`#${selector}`) !== null;
}

export function useWheelActions({
    throttleMs = 300,
    minDelta = 20,
}: {
    throttleMs?: number;
    minDelta?: number;
} = {}) {
    const pendingAsync = useRef(0);
    const gameProps = useGameProps();

    const runAsync = async (
        fn: (props: StepLabelProps) => Promise<unknown>,
    ) => {
        try {
            pendingAsync.current += 1;
            GameStatus.setLoading(pendingAsync.current > 0);
            await fn(gameProps);
        } finally {
            pendingAsync.current -= 1;
            GameStatus.setLoading(pendingAsync.current > 0);
            if (pendingAsync.current === 0) {
                gameProps.invalidateInterfaceData();
            }
        }
    };

    const handleWheel = useThrottler(
        async (event: WheelEvent) => {
            if (
                !(
                    isInsideRoot(event.target, HTML_UI_LAYER_NAME) ||
                    isInsideRoot(event.target, HTML_CANVAS_LAYER_NAME)
                )
            )
                return;
            if (hasScrollableParent(event.target)) return;

            // blocca lo scroll nativo
            event.preventDefault();

            const { deltaY } = event;

            // ignora micro-movimenti del trackpad
            if (Math.abs(deltaY) < minDelta) return;

            if (deltaY < 0) {
                // ⬆️ Scroll up
                await runAsync(narration.continue.bind(narration));
            }

            if (deltaY > 0) {
                // ⬇️ Scroll down
                await runAsync(stepHistory.back.bind(stepHistory));
            }
        },
        { wait: throttleMs },
    );

    useEffect(() => {
        window.addEventListener("wheel", handleWheel.maybeExecute, {
            passive: false,
        });

        return () => {
            window.removeEventListener("wheel", handleWheel.maybeExecute);
        };
    }, [handleWheel.maybeExecute]);

    return null;
}
```
