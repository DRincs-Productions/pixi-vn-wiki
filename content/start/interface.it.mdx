**Cos'è il UI?** UI è l'acronimo di User Interface (interfaccia utente). Si riferisce agli elementi visivi e ai componenti interattivi attraverso i quali un utente interagisce con un dispositivo digitale, un'applicazione o un sito web. L'interfaccia utente include tutto ciò con cui gli utenti interagiscono, come pulsanti, menu, icone, campi di testo e layout. L'obiettivo della progettazione dell'interfaccia utente è creare un'interfaccia che sia al tempo stesso esteticamente gradevole e funzionale, offrendo agli utenti un'esperienza fluida e intuitiva.

Pixi’VN non offre alcun componente per creare l’interfaccia utente del gioco. Per come è stato progettato, l'interfaccia utente deve essere creata utilizzando framework JavaScript esterni. Ciò consente l'utilizzo di sistemi come React, Vue, ecc. per creare **schermate UI** molto più complesse con prestazioni eccellenti.

<Accordion title="Adding HTML UI Layers" id="adding-html-ui-layers">

**What is an HTML UI Layer?** An HTML UI Layer is an HTML element that is added above the PixiJS canvas and has the same dimensions as the canvas. This allows you to create a UI with a JavaScript framework such as React or Vue.

To add an HTML UI Layer, you can use the `canvas.addHtmlLayer` method. This method takes the following parameters:

- `id`: the id of the layer, it must be unique.
- `element`: the HTML element to add as a layer.
- `style`: an object with the style properties to apply to the layer. The default style is `{ position: "absolute", pointerEvents: "none" }`.

Ad esempio:

```ts tab="src/main.ts"
const root = document.getElementById("root");
if (!root) {
    throw new Error("root element not found");
}
const htmlLayer = canvas.addHtmlLayer("ui", root, {
    position: "absolute",
    pointerEvents: "none"
});
```

```html tab="index.html"
<!doctype html>
<html lang="en">
  <body>
    <div id="root"></div>
    <script type="module" src="/src/main.ts"></script>
  </body>
</html>
```

To get the HTML element of the layer, you can use the `canvas.getHtmlLayer` method. This method takes the following parameter:

- `id`: the id of the layer.

To remove an HTML UI Layer, you can use the `canvas.removeHtmlLayer` method. This method takes the following parameter:

- `id`: the id of the layer.

</Accordion>

<Accordion title="How to enable HTML UI interaction?" id="how-to-enable-ui-interaction">

By default, all HTML elements of the HTML UI Layers have the `pointer-events: none` style. This means that the HTML elements will not intercept any mouse or touch events, allowing the PixiJS canvas to receive all events. This is useful when you want to display an image or a video in the canvas and you don't want the UI to interfere with it.

So you must set the `pointer-events: auto` style only for the conponent (example a button, a form, etc...) that you want to interact with the user.

Ad esempio:

```tsx tab="React"
export default function NextButton() {
    return (
        <button
            style={{
                pointerEvents: "auto",
            }}
        >
            Next
        </button>
    );
}
```

```vue tab="Vue"
<template>
    <button style="pointer-events: auto;">
        Next
    </button>
</template>
```

</Accordion>

<Accordion title="Adding PixiJS UI Layers" id="adding-pixijs-ui-layers">

**What is an PixiJS UI Layer?** It is PixiJS Container. Unlike the Container created and used by Pixi’VN:

- Your current state will not be saved in any way.
- It has better performance. Pixi’VN will not do any checks on it.
- You can't use Pixi'VN features.
- Being a core PixiJS component, you can add all PixiJS compatible components.

To add a PixiJS UI Layer, you can use the `canvas.addLayer` method. This method takes the following parameters:

- `label`: the PixiJS Container label, it must be unique.

Ad esempio:

```ts title="src/main.ts"
const container = canvas.addLayer("ui");
```

To get the PixiJS Container of the layer, you can use the `canvas.getLayer` method. This method takes the following parameter:

- `label`: the PixiJS Container label.

To remove a PixiJS UI Layer, you can use the `canvas.removeLayer` method. This method takes the following parameter:

- `label`: the PixiJS Container label.

## Differenze tra l'interfaccia utente e il canvas

Così come è stato progettato e concepito Pixi’VN, l’interfaccia utente e la tela sono due elementi distinti e indipendenti. L'interfaccia utente si trova sopra la tela e viene utilizzata per creare pulsanti, moduli, ecc. La tela viene utilizzata per visualizzare immagini, video, ecc.

All information about the current state of the canvas is included in the save and it is possible to restore the state of a previous step. The current state of the UI will not be included in the saves. So you have to <DynamicLink href="/start/interface-connect-storage">manage it yourself by saving the information</DynamicLink> you need to <DynamicLink href="/start/storage">game storage</DynamicLink> or browser storage.

In the canvas you can add components during each step. In the UI you can't do that, you can create several <DynamicLink href="/start/interface-connect-storage">"screens" and navigate between them</DynamicLink>.

In the canvas you can only add PixiJS components, they are usually composed of images and are very simple. In the UI you can add any HTML element or use any UI component library, so you can create much more complex interfaces.
