**Cos'è il UI?** UI è l'acronimo di User Interface (interfaccia utente). Si riferisce agli elementi visivi e ai componenti interattivi attraverso i quali un utente interagisce con un dispositivo digitale, un'applicazione o un sito web. L'interfaccia utente include tutto ciò con cui gli utenti interagiscono, come pulsanti, menu, icone, campi di testo e layout. L'obiettivo della progettazione dell'interfaccia utente è creare un'interfaccia che sia al tempo stesso esteticamente gradevole e funzionale, offrendo agli utenti un'esperienza fluida e intuitiva.

Pixi’VN does not provide built-in components to create the game UI. Instead, you should use external JavaScript frameworks to build your UI. This allows you to leverage systems such as React, Vue, etc., to create complex and high-performance **UI screens**.

<Accordion title="Adding HTML UI Layers" id="adding-html-ui-layers">

**Cos'è un HTML UI Layer?** Un HTML UI Layer è un elemento HTML che viene aggiunto sopra il canvas di PixiJS e ha le stesse dimensioni del canvas. Ciò consente di creare un'interfaccia utente con un framework JavaScript come React o Vue.

To add an HTML UI Layer, you can use the `canvas.addHtmlLayer` function. Questa funzione ha i seguenti parametri:

- `id`: The id of the layer, it must be unique.
- `element`: the HTML element to add as a layer.
- `style`: An object with the style properties to apply to the layer. The default style is `{ position: "absolute", pointerEvents: "none" }`.

Ad esempio:

```ts tab="src/main.ts"
const root = document.getElementById("root");
if (!root) {
    throw new Error("root element not found");
}
const htmlLayer = canvas.addHtmlLayer("ui", root, {
    position: "absolute",
    pointerEvents: "none"
});
```

```html tab="index.html"
<!doctype html>
<html lang="en">
  <body>
    <div id="root"></div>
    <script type="module" src="/src/main.ts"></script>
  </body>
</html>
```

To get the HTML element of the layer, you can use the `canvas.getHtmlLayer` function. Questa funzione ha i seguenti parametri:

- `id`: The id of the layer.

To remove an HTML UI Layer, you can use the `canvas.removeHtmlLayer` function. Questa funzione ha i seguenti parametri:

- `id`: The id of the layer.

<Accordion title="How to enable HTML UI interaction?" id="how-to-enable-ui-interaction">

Per impostazione predefinita, tutti gli elementi HTML dei HTML UI Layers hanno lo stile `pointer-events: none`. Ciò significa che gli elementi HTML non intercetteranno alcun evento del mouse o del tocco, consentendo al canvas di PixiJS di ricevere tutti gli eventi. Questa funzionalità è utile quando si desidera visualizzare un'immagine o un video nell'area di lavoro e non si desidera che l'interfaccia utente interferisca con essi.

So you must set the `pointer-events: auto` style only for the conponent that you want to interact with the user.

Ad esempio:

```tsx tab="React"
export default function NextButton() {
    return (
        <button
            style={{
                pointerEvents: "auto",
            }}
        >
            Next
        </button>
    );
}
```

```vue tab="Vue"
<template>
    <button style="pointer-events: auto;">
        Next
    </button>
</template>
```

</Accordion>

<Accordion title="Adding PixiJS UI Layers" id="adding-pixijs-ui-layers">

**Cos'è un PixiJS UI Layer?** È un PixiJS Container. Unlike the main Container dedicated only for Pixi’VN features:

- Your current state will not be included in saves.
- It has better performance. Pixi’VN will not do any checks on it.
- You can't use Pixi'VN features.
- Being a core PixiJS component, you can add all PixiJS compatible components.

To add a PixiJS UI Layer, you can use the `canvas.addLayer` function. Questa funzione ha i seguenti parametri:

- `label`: The PixiJS Container label, it must be unique.

Ad esempio:

```ts title="src/main.ts"
const container = canvas.addLayer("ui");
```

To get the PixiJS Container of the layer, you can use the `canvas.getLayer` function. Questa funzione ha i seguenti parametri:

- `label`: The PixiJS Container label.

To remove a PixiJS UI Layer, you can use the `canvas.removeLayer` function. Questa funzione ha i seguenti parametri:

- `label`: The PixiJS Container label.

## Differenze tra l'interfaccia utente e il canvas

Così come è stato progettato e concepito Pixi’VN, l’interfaccia utente e la tela sono due elementi distinti e indipendenti. L'interfaccia utente si trova sopra la tela e viene utilizzata per creare pulsanti, moduli, ecc. La tela viene utilizzata per visualizzare immagini, video, ecc.

Tutte le informazioni sullo stato attuale del canvas sono incluse nel salvataggio ed è possibile ripristinare lo stato di un passaggio precedente. Lo stato attuale dell'interfaccia utente non verrà inclusa nei salvataggi. Quindi devi <DynamicLink href="/start/interface-connect-storage">gestirlo tu stesso salvando le informazioni</DynamicLink> di cui hai bisogno nella <DynamicLink href="/start/storage">memoria di gioco</DynamicLink> o nella memoria del browser.

Nel canvas è possibile aggiungere componenti durante ogni step. Nell'interfaccia utente non è possibile farlo, è possibile creare diverse <DynamicLink href="/start/interface-connect-storage">"schermate" e navigare tra di esse</DynamicLink>.

In the canvas you can only add Pixi’VN components. In the UI you can add any HTML/PixiJS component or use any UI component library, so you can create much more complex interfaces.
