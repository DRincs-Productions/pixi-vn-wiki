---

비주얼 노벨은 일반적으로 이미지와 텍스트가 포함된 일련의 장면으로 구성됩니다. Pixi’VN에서는 전체 내러티브 흐름이 내러티브 노드 (`label`)와 내러티브 단계 (`step`)의 개념을 기반으로 합니다.

내러티브 노드 (`label`)는 내러티브를 구성하는 데 사용되며, 이야기의 "북마크" 또는 섹션으로 기능합니다. 기술적으로, 내러티브 노드 (`label`)는 내러티브 단계 (`step`)의 컨테이너 역할을 하는 클래스입니다.

내러티브 단계 (`step`)는 내러티브 노드 (`label`) 내에서 발생하는 개별 동작 또는 이벤트입니다. 스텝은 이미지, 텍스트 및 기타 내러티브 요소를 표시하는 데 사용됩니다.

:::tip[용어 설명]

내러티브 노드 (`label`)라는 용어는 Ren'Py에서 가져온 것으로, 이야기의 "북마크" 또는 "랜드마크" 역할을 합니다. ink 언어에서 동등한 용어는 내러티브 노드 (`knot`)입니다.내러티브 단계 (`step`)라는 용어는 내러티브 노드 (`label`) 내에서 발생하는 개별 동작이나 이벤트를 나타내기 위해 Pixi’VN에서 도입한 용어입니다.

내러티브 단계 (`step`)라는 용어는 내러티브 노드 (`label`) 내에서 발생하는 개별 동작이나 이벤트를 나타내기 위해 Pixi’VN에서 도입한 용어입니다.

:::

<Accordions>

<Accordion title="생명 주기" id="lifecycle">

1. 게임은 [`Game.start`](/jsdoc/pixi-vn/index/namespaces/Game/functions/start) 함수로 내러티브 노드 (`label`)를 호출하여 시작됩니다. 내러티브 노드의 첫 번째 내러티브 단계 (`step`)가 자동으로 실행됩니다.
2. 그 후, [`narration.continue`](/jsdoc/pixi-vn/index/interfaces/NarrationManagerInterface#continue) 함수를 이벤트(예: 버튼 클릭)에 연결하여, 함수가 호출될 때마다 내러티브 노드 (`label`)의 다음 내러티브 단계 (`step`)가 실행됩니다. 일부 내러티브 단계 (`step`)는 다른 내러티브 노드 (`label`)를 시작할 수도 있습니다.
3. 모든 내러티브 단계 (`step`)가 완료될 때만 게임이 종료됩니다. 그 시점에 [`Game.onEnd`](/jsdoc/pixi-vn/index/namespaces/Game/functions/onEnd) 함수가 트리거됩니다.

</Accordion>

</Accordions>

## 추가

내러티브 노드 (`label`)를 생성하려면 [`newLabel`](/jsdoc/pixi-vn/index/functions/newLabel) 함수를 사용하세요.

```ts title="content/labels/start.label.ts"
import { narration, StepLabelProps } from "@drincs/pixi-vn";

export const startLabel = newLabel("start", [
    () => {
        narration.dialogue = { character: liam, text: "Example of dialogue" };
    },
    (props: StepLabelProps, { labelId }) => narration.jump(labelId, props),
]);
```

## 기타 기능

<Accordions>

<Accordion title="`StepLabelProps`" id="steplabelprops">

:::tip[템플릿]

모든 템플릿에서 `StepLabelProps` 인터페이스는 이미 재정의되어 있습니다. `pixi-vn.d.ts` 파일에서 확인할 수 있습니다.

:::

모든 <DynamicLink href="/start/labels-flow">내러티브 흐름 제어 함수</DynamicLink>는 [`StepLabelProps`](/jsdoc/pixi-vn/index/interfaces/StepLabelProps) 인터페이스에 맞는 객체를 필요로 합니다. `.d.ts` 파일에서 `StepLabelProps` 인터페이스를 "재정의"하여 필수 매개변수를 지정할 수 있습니다.

```ts tab="pixi-vn.d.ts"
import { narration } from "@drincs/pixi-vn";

declare module "@drincs/pixi-vn" {
    interface StepLabelProps {
        navigate: (route: string) => void;
    }
}
```

```ts tab="content/labels/start.label.ts"
export const startLabel = newLabel("start", [
    ({ navigate }) => {
        navigate("/new-route");
    },
]);
```

</Accordion>

<Accordion title="`step` 속성" id="step-pros">

[`newLabel`](/jsdoc/pixi-vn/index/functions/newLabel) 함수에 타입을 전달하여 [`StepLabelProps`](#steplabelprops) 외에 해당 내러티브 노드 (`label`)에 대한 사용자 정의 매개변수를 추가할 수 있습니다.

```ts title="content/labels/start.label.ts"
import { narration } from "@drincs/pixi-vn";

export const startLabel = newLabel<{ name: string }>("start", [
    (props) => {
        console.log(props.name);
    },
]);

narration.call(startLabel, {
    // Add StepLabelProps here
    navigate: navigate, // example
    // And the custom props for the label
    name: "John",
});
```

</Accordion>

<Accordion title="`StepLabelResult`" id="step-results">

내러티브 단계는 [`StepLabelResult`](/jsdoc/pixi-vn/index/interfaces/StepLabelResult) 객체를 반환할 수 있습니다. `.d.ts` 파일에서 `StepLabelResult` 인터페이스를 "재정의"하여 내러티브 단계 (`step`) 결과에 대한 사용자 정의 속성을 정의할 수 있습니다.

```ts tab="pixi-vn.d.ts"
declare module "@drincs/pixi-vn" {
    interface StepLabelResult {
        newRoute?: string;
        [key: string]: any;
    }
}
```

```ts tab="content/labels/start.label.ts"
export const startLabel = newLabel("start", [
    () => {
        return {
            newRoute: "/new-route",
            customProperty: 12,
        };
    },
]);
```

</Accordion>
