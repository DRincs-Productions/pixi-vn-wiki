Pixi’VN fournit un ensemble de composants canvas. Ces composants étendent les composants [Pixi.js](https://pixijs.com/) en ajoutant diverses fonctionnalités, comme la capacité de sauvegarder leur état actuel.

<Accordions>

<Accordion title="Composants intégrés" id="base-components">

* [`Sprite`](/jsdoc/pixi-vn/index/classes/Sprite) correspond au composant [`PixiJS.Sprite`](https://pixijs.com/8.x/guides/components/sprites).
* [`Container`](/jsdoc/pixi-vn/index/classes/Container) correspond au composant [`PixiJS.Container`](https://pixijs.com/8.x/guides/components/containers).
* <DynamicLink href="/start/canvas-image">`ImageSprite`</DynamicLink> est un composant introduit par Pixi’VN.
* <DynamicLink href="/start/canvas-image-container">`ImageContainer`</DynamicLink> est un composant introduit par Pixi’VN.
* <DynamicLink href="/start/canvas-video">`VideoSprite`</DynamicLink> est un composant introduit par Pixi’VN.
* <DynamicLink href="/start/canvas-text">`Text`</DynamicLink> est un composant introduit par Pixi’VN.

</Accordion>

<Accordion title="Composants de plugin" id="additional-components">

* <DynamicLink href="/start/canvas-spine2D">Spine 2D</DynamicLink> est un composant installable via le système de plugins de Pixi’VN. Il est utilisé pour afficher les animations Spine 2D.

## Autres fonctionnalités

<Accordions>

<Accordion title="Composants personnalisés" id="custom-components">

Vous pouvez créer des composants personnalisés en étendant les composants de base. Pour ce faire, vous devez utiliser le décorateur [`@canvasComponentDecorator`](/jsdoc/pixi-vn/index/functions/canvasComponentDecorator).

Il est nécessaire de surcharger le getter [`memory`](/jsdoc/pixi-vn/index/classes/CanvasBaseItem#memory) et la méthode [`setMemory`](/jsdoc/pixi-vn/index/classes/CanvasBaseItem#setmemory) afin de stocker les propriétés du composant personnalisé.
Dans `get memory()`, il est très important de retourner la propriété `className` ; cette propriété doit être égale à l'identifiant utilisé dans le décorateur.

Par exemple, vous pouvez créer une classe `AlienTinting` qui étend la classe `Sprite` pour gérer la direction et la vitesse de chaque alien individuel dans une animation.

```ts title="canvas/components/AlienTinting.ts"
const ALIEN_TINTING_TEST_ID = "AlienTintingTest";
@canvasComponentDecorator({
    name: ALIEN_TINTING_TEST_ID,
})
class AlienTintingTest extends Sprite<IAlienTintingMemory> {
    readonly pixivnId: string = CANVAS_SPINE_ID;
    override get memory() {
        return {
            ...super.memory,
            pixivnId: CANVAS_SPINE_ID,
            direction: this.direction,
            turningSpeed: this.turningSpeed,
            speed: this.speed,
        };
    }
    override async setMemory(memory: IAlienTintingMemory) {
        await super.setMemory(memory);
        this.direction = memory.direction;
        this.turningSpeed = memory.turningSpeed;
        this.speed = memory.speed;
    }
    direction: number = 0;
    turningSpeed: number = 0;
    speed: number = 0;
    static override from(
        source: Texture | TextureSourceLike,
        skipCache?: boolean,
    ) {
        let sprite = Sprite.from(source, skipCache);
        let mySprite = new AlienTintingTest();
        mySprite.texture = sprite.texture;
        return mySprite;
    }
}
```
