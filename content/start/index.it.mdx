Puoi iniziare a utilizzare Pixi'VN [inizializzando un nuovo progetto](#project-initialization) o [installando il pacchetto](#installation) in un progetto esistente.

## Prerequisiti

Prima di iniziare, è necessario avere installato i seguenti strumenti:

- [Node.js](https://nodejs.org/) versione 18 o superiore.
- Editor di testo con supporto per TypeScript, come:
  - [Visual Studio Code](https://code.visualstudio.com/)
  - [Cursor](https://www.cursor.com/)
  - [VSCodium](https://vscodium.com/)
- (Raccomandato) [Git](https://git-scm.com/)
  - Un [account GitHub](https://github.com/)

## Inizializzazione del progetto

Se vuoi iniziare da un nuovo progetto, puoi usare il seguente comando per inizializzare un nuovo progetto con i template Pixi'VN:

```sh tab="npm"
npm create pixi-vn@latest
```

```sh tab="yarn"
yarn create pixi-vn
```

```sh tab="pnpm"
pnpm create pixi-vn
```

```sh tab="bun"
bun create pixi-vn
```

```sh tab="deno"
deno init --npm pixi-vn
```

Puoi vedere l'elenco dei template disponibili e delle demo interattive <DynamicLink href="/start/templates">qui</DynamicLink>.

Dopo aver inizializzato il progetto, apri la directory del progetto con il tuo editor di testo (si consiglia VSCode) e inizia a sviluppare il progetto.

Tutti i templates includono un file `README.md` con maggiori informazioni sul progetto.

## Installazione

Per installare il pacchetto Pixi'VN in un progetto JavaScript esistente, utilizzare uno dei seguenti comandi:

```sh tab="npm"
npm install @drincs/pixi-vn
```

```sh tab="yarn"
yarn add @drincs/pixi-vn
```

```sh tab="pnpm"
pnpm add @drincs/pixi-vn
```

```sh tab="bun"
bun add @drincs/pixi-vn
```

```sh tab="deno"
deno install npm:@drincs/pixi-vn
```

<Accordion title="TypeScript problem" id="ts-installation">

In some TypeScript projects, you might encounter issues when importing the Pixi’VN package. This is because the package is made up of submodules, and until recently TypeScript ignored "exports" unless you used `moduleResolution`: `nodenext` or `bundler`.

To solve this problem, if your project is of type `module` (i.e. `type: "module"` in your `package.json`), you can add the following lines to your `tsconfig.json` file:

```json
{
    "compilerOptions": {
        "module": "esnext",
        "moduleResolution": "bundler"
    }
}
```

If your project is of type `commonjs`, you can add the following lines to your `tsconfig.json` file:

```json
{
    "compilerOptions": {
        "module": "nodenext",
        "moduleResolution": "nodenext"
    }
}
```

Ora devi [inizializzare Pixi’VN](#project-initialization) prima di utilizzare l'engine. Ad esempio, puoi aggiungere il seguente codice a `main.ts` o `index.ts` (a seconda della configurazione del progetto).

### Inizializzare

Prima di utilizzare l'engine Pixi'VN, è necessario inizializzare il gioco. Puoi farlo chiamando il metodo `Game.init`.

Questa funzione ha i seguenti parametri:

- `element`: L'elemento HTML a cui aggiungere il canvas.
- `options`: Equivalente alle opzioni che puoi utilizzare durante l'inizializzazione di un'[applicazione PixiJS](https://pixijs.com/8.x/guides/basics#creating-an-application). Le seguenti opzioni sono obbligatorie:
  - `width`: La larghezza del canvas.
  - `height`: L'altezza del canvas.
- `devtoolsOptions`: Equivalente alle opzioni che puoi utilizzare durante l'inizializzazione di <DynamicLink href="/start/canvas#use-pixijs-devtools-with-pixivn">PixiJS Devtools</DynamicLink>.

```ts title="src/main.tsx"
import { Game } from "@drincs/pixi-vn";

// Canvas setup with PIXI
const body = document.body
if (!body) {
    throw new Error('body element not found')
}

Game.init(body, {
    height: 1080,
    width: 1920,
    backgroundColor: "#303030",
}).then(() => {
    // ...
});

// read more here: https://pixi-vn.web.app/start/other-narrative-features.html#how-manage-the-end-of-the-game
Game.onEnd(async (props) => {
    Game.clear();
    props.navigate("/");
});

Game.onError((type, error, { notify }) => {
    notify("allert_error_occurred");
});
```
