キャンバスに追加するすべてのコンポーネントには、エイリアス（`alias`）を割り当てる必要があります。 エイリアス（`alias`）とは、一意の文字列でコンポーネントを参照するための手段です。

<Accordions>

<Accordion title="`alias` は PixiJS で何に対応しますか？" id="what-is-an-alias">

エイリアス（`alias`）は `PixiJSComponent.label` に対応しています。そのため `label` を直接変更せず、<DynamicLink href="/start/canvas-functions">Pixi’VN が提供するメソッド</DynamicLink> を使用してエイリアス（`alias`）を変更してください。

</Accordion>

</Accordions>

既存のエイリアス（`alias`）を指定してコンポーネントを追加すると、新しいコンポーネントが古いコンポーネントを置き換えます。

## 継承ファクター

既存のエイリアス（`alias`）を使用してコンポーネントを追加すると、新しいコンポーネントは古いコンポーネントを置き換えるだけでなく、古いコンポーネントのプロパティ、`zIndex`、および <DynamicLink href="/start/canvas-tickers">tickers</DynamicLink> を継承します。

## 編集

キャンバスコンポーネントのエイリアス（`alias`）を編集するには、[`canvas.editAlias`](/jsdoc/pixi-vn/index/interfaces/CanvasManagerInterface#editalias) を使用できます。 そのエイリアス（`alias`）に1つ以上の <DynamicLink href="/start/canvas-tickers">tickers</DynamicLink> が関連付けられている場合、ticker 内でも自動的に更新されます。

```ts
import { canvas } from "@drincs/pixi-vn";

canvas.editAlias("sprite1", "sprite2");
```

## その他の機能

<Accordions>

<Accordion title="ゲームレイヤーのエイリアス" id="game-layer-alias">

Pixi’VN において、ゲームレイヤーはすべてのゲーム要素がレンダリングされるメインゲームエリアを表す特殊なコンポーネントです。

このコンポーネントには特別なエイリアス（`alias`）として [`CANVAS_APP_GAME_LAYER_ALIAS`](/jsdoc/pixi-vn/index/variables/CANVAS_APP_GAME_LAYER_ALIAS) が割り当てられており、スクリプト内でゲームレイヤーを参照するために使用されます。

これは、レイヤー全体に対して <DynamicLink href="/start/canvas-animations-effects">アニメーションやエフェクト</DynamicLink> を適用したい場合に非常に便利です。 削除など、このアイテムには許可されていない機能もあります。

```ts
import { CANVAS_APP_GAME_LAYER_ALIAS, shakeEffect } from "@drincs/pixi-vn";

shakeEffect(CANVAS_APP_GAME_LAYER_ALIAS);
```
