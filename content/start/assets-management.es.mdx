---
title: Carga de assets
description: Aprende a cargar y gestionar assets en Pixi’VN usando alias, bundles y diferentes estrategias de carga para mantener tu juego fluido.
---

Especialmente en juegos con assets en línea, seguir buenas prácticas de carga es esencial para evitar largos tiempos de espera y mantener una experiencia fluida para el jugador.

Para cargar y manipular assets (imágenes, gifs, vídeos, etc.) necesitarás usar `Assets`. `Assets` es una clase con muchas funcionalidades y proviene de la librería PixiJS. Para más información, lee [aquí](https://pixijs.com/8.x/guides/components/assets).

## Usar alias para cargar assets

Para cargar un asset, usa siempre su **alias** — el identificador único definido en el manifiesto o al registrar el asset. Se desaconseja firmemente usar la ruta `src` directamente: acopla tu código a la ubicación del archivo, haciéndolo frágil cuando cambien las rutas o el alojamiento.

```ts title="main.ts"
import { Assets } from "@drincs/pixi-vn";

const texture = await Assets.load("eggHead");
```

```json title="assets/manifest.gen.json"
{
    "bundles": [
        {
            "name": "default",
            "assets": [
                {
                    "alias": "eggHead",
                    "src": "./assets/eggHead.png"
                }
            ]
        }
    ]
}
```

## Organizar los assets en bundles

Una buena práctica es agrupar los assets en **bundles** en lugar de registrarlos individualmente. Los bundles te permiten cargar un conjunto completo de assets con una sola llamada, y diferir la carga hasta que los assets sean realmente necesarios.

:::tip

Para los assets locales, nombrar los bundles en función de dónde se usan es menos crítico — dado que los archivos locales se cargan instantáneamente, no hay ningún beneficio real en dividirlos por pantalla o `label`. Esta convención es más valiosa para los assets en línea, donde una carga dirigida reduce directamente los tiempos de espera.

:::

Se recomienda nombrar cada bundle en función de **dónde se usará**. Por ejemplo:

* usa el id del `label` (por ejemplo, `startLabel.id`) como nombre del bundle para los assets usados en ese `label` específico
* usa la ruta de la ruta (por ejemplo, `"/"`) como nombre del bundle para los assets usados en la pantalla correspondiente, como el menú principal

De esta forma, puedes cargar exactamente los assets correctos en el momento adecuado, sin cargar más de lo necesario.

```ts title="assets/index.ts"
import { startLabel } from "@/content/labels/start.label";
import type { FileRouteTypes } from "@/routeTree.gen";
import type { AssetsManifest } from "@drincs/pixi-vn";

export const manifest: AssetsManifest = {
    bundles: [
        // screens
        {
            // main menu
            name: "/" as FileRouteTypes["fullPaths"],
            assets: [
                {
                    alias: "background_main_menu",
                    src: "https://raw.githubusercontent.com/user/project/refs/heads/main/main-menu.png",
                },
            ],
        },
        // labels
        {
            name: startLabel.id,
            assets: [
                {
                    alias: "bg01-hallway",
                    src: "https://raw.githubusercontent.com/user/project/refs/heads/main/breakdown/bg01-hallway.webp",
                },
            ],
        },
    ],
};
```

## Cuándo cargar los assets

Por defecto, los assets se cargan bajo demanda — solo cuando son realmente necesarios. Sin embargo, cargar en el último momento puede causar pausas notables durante el juego.

Estos son los momentos más comunes y recomendados para cargar los assets con antelación:

<Accordions>

<Accordion title="Cargar assets al inicio del proyecto" id="load-assets-at-project-start">

Es posible cargar assets al inicio del proyecto antes de que el jugador pueda interactuar con él, por ejemplo durante la pantalla de carga inicial. Se sugiere usar este procedimiento solo para los assets utilizados en la página principal o para los assets que se usan con frecuencia, y no superar los 100 MB.

Para ello, debes usar la función `Assets.load` y esperar a que finalice (con `await`) cuando el proyecto se inicia.

```ts tab="lib/utils/assets-utility.ts"
import { manifest } from "@/assets";
import { Assets, sound } from "@drincs/pixi-vn";

let assetsInitialized = false;

/**
 * Define all the assets that will be used in the game.
 * This function will be called before the game starts.
 */
export async function defineAssets() {
    if (!assetsInitialized) {
        await Assets.init({ manifest });
        assetsInitialized = true;
    }

    // The game will not start until these asserts are loaded.
    await Assets.loadBundle("base");
}
```

```ts tab="assets/index.ts"
import type { AssetsManifest } from "@drincs/pixi-vn";

export const manifest: AssetsManifest = {
    bundles: [
        {
            name: "base",
            assets: [
                {
                    alias: "eggHead",
                    src: "https://pixijs.com/assets/eggHead.png",
                },
                {
                    alias: "flowerTop",
                    src: "https://pixijs.com/assets/flowerTop.png",
                },
            ],
        },
    ],
};
```

</Accordion>

<Accordion title="Cargar assets en segundo plano al inicio del proyecto" id="load-assets-in-the-background-at-project-start">

Es posible cargar assets en segundo plano cuando el proyecto se inicia, reduciendo los tiempos de carga al arrancar el proyecto. Se recomienda no superar los 2 GB.

Para ello, puedes usar la función `Assets.backgroundLoad` cuando el proyecto se inicia. Puedes obtener más información sobre la función `Assets.backgroundLoad` [aquí](https://pixijs.com/8.x/guides/components/assets).

```ts tab="lib/utils/assets-utility.ts"
import { manifest } from "@/assets";
import { Assets, sound } from "@drincs/pixi-vn";

let assetsInitialized = false;

/**
 * Define all the assets that will be used in the game.
 * This function will be called before the game starts.
 */
export async function defineAssets() {
    if (!assetsInitialized) {
        await Assets.init({ manifest });
        assetsInitialized = true;
    }

    // The game will start immediately, but these assets will be loaded in the background.
    Assets.backgroundLoadBundle(["audio"]);
}
```

```ts tab="assets/index.ts"
import type { AssetsManifest } from "@drincs/pixi-vn";

export const manifest: AssetsManifest = {
    bundles: [
        {
            name: "audio",
            assets: [
                {
                    alias: "bgm",
                    src: "https://raw.githubusercontent.com/user/project/refs/heads/main/audio/bgm.mp3",
                },
            ],
        },
    ],
};
```

</Accordion>

<Accordion title="Cargar assets al inicio de un label" id="load-assets-at-label-start">

:::tip[*ink*]

Puedes usar este método con la sintaxis *ink*. Ver más <DynamicLink href="/ink/assets">aquí</DynamicLink>.

:::

Para agrupar las cargas de un `step` a otro de un `label` en una sola carga, es posible cargar todos o parte de los assets usados por el `label` antes de que este comience. Dado que esto no reduce los tiempos de espera para el jugador, sino que los concentra en una sola carga, se recomienda [cargarlos en segundo plano](#load-assets-in-the-background-at-label-start).

Para ello, debes usar la función `Assets.load` y esperar a que finalice (con `await`) cuando el `label` se inicia. Usarás la función <DynamicLink href="/start/labels-advanced#onloadinglabel">`onLoadingLabel`</DynamicLink>.

Para un código más limpio, se recomienda definir un bundle en tu manifiesto con el id correspondiente al id del `label` y definir dentro de él los assets que se usarán en ese `label`.

```ts tab="content/labels/start.label.ts"
import { newLabel, showImage, Assets } from "@drincs/pixi-vn";

const startLabel = newLabel(
    "start",
    [
        () => {
            await showImage("eggHead");
        },
        () => {
            await showImage("flowerTop");
        },
    ],
    {
        onLoadingLabel: async (_stepId, label) => {
            // The label will not start until these assets are loaded.
            await Assets.loadBundle(label.id);
    },
);
export default startLabel;
```

```ts tab="assets/index.ts"
import type { AssetsManifest } from "@drincs/pixi-vn";
import startLabel from "@/content/labels/start.label";

export const manifest: AssetsManifest = {
    bundles: [
        {
            name: startLabel.id,
            assets: [
                {
                    alias: "eggHead",
                    src: "https://pixijs.com/assets/eggHead.png",
                },
                {
                    alias: "flowerTop",
                    src: "https://pixijs.com/assets/flowerTop.png",
                },
            ],
        },
    ],
};
```

</Accordion>

<Accordion title="Cargar assets en segundo plano al inicio de un label" id="load-assets-in-the-background-at-label-start">

:::tip[Plantillas]

En todas las plantillas, cuando se inicia un `label`, se inicia la carga en segundo plano del bundle (si existe) con el id correspondiente al `label`. Por lo tanto, es suficiente con **crear un manifiesto con un bundle para cada `label`**.

:::

Para hacer el juego más fluido intentando eliminar los tiempos de carga de assets de un `step` a otro, es posible iniciar un "grupo de carga" al comienzo de un `label`. Esto significa que potencialmente no percibirás ninguna carga, especialmente en los `steps` posteriores del `label`.

Para ello, puedes usar la función `Assets.backgroundLoad` cuando el `label` se inicia. Puedes obtener más información sobre la función `Assets.backgroundLoad` [aquí](https://pixijs.com/8.x/guides/components/assets). Y usarás la función <DynamicLink href="/start/labels-advanced#onloadinglabel">`onLoadingLabel`</DynamicLink>.

Para un código más limpio, se recomienda definir un bundle en tu manifiesto con el id correspondiente al id del `label` y definir dentro de él los assets que se usarán en ese `label`.

```ts tab="content/labels/start.label.ts"
import { newLabel, showImage, Assets } from "@drincs/pixi-vn";

const startLabel = newLabel(
    "start",
    [
        () => {
            await showImage("eggHead");
        },
        () => {
            await showImage("flowerTop");
        },
    ],
    {
        onLoadingLabel: async (_stepId, label) => {
            // The label will start immediately, but these assets will be loaded in the background.
            Assets.backgroundLoadBundle(label.id);
        },
    },
);
export default startLabel;
```

```ts tab="assets/index.ts"
import type { AssetsManifest } from "@drincs/pixi-vn";

            ],
        },
    ],
};
```

<Accordion title="Cargar assets antes de mostrar una pantalla" id="load-assets-before-screen">

Es posible cargar assets antes de que se muestre una pantalla. Esto es útil cuando una pantalla usa imágenes u otros recursos que deben estar listos antes de que el componente se renderice.

En este ejemplo se usa TanStack Router, que ya está incluido en todas las plantillas. Puedes encontrar más información al respecto <DynamicLink href="/start/interface-navigate">aquí</DynamicLink>.

La función `loader` se ejecuta antes de que el componente de la ruta se monte. Puedes llamar a `Assets.loadBundle` dentro de `loader` para precargar el bundle asociado a esa ruta.

Además, puedes usar `pendingComponent` para mostrar un indicador de carga mientras se obtienen los assets, de modo que el usuario vea una transición significativa en lugar de una pantalla en blanco.

```tsx tab="routes/game/narration.tsx"
import { NarrationPendingComponent } from "@/components/loading";
import { NarrationScreen } from "@/components/screens/narration";
import { Assets } from "@drincs/pixi-vn";
import { createFileRoute } from "@tanstack/react-router";

export const Route = createFileRoute("/game/narration")({
    component: NarrationElement,
    pendingComponent: NarrationPendingComponent,
    loader: async () => {
        await Assets.loadBundle("/game/narration");
    },
});

function NarrationElement() {
    return <NarrationScreen />;
}
```

```ts tab="assets/index.ts"
import type { AssetsManifest } from "@drincs/pixi-vn";
import startLabel from "@/content/labels/start.label";

export const manifest: AssetsManifest = {
    bundles: [
        {
            name: "/game/narration",
            assets: [
                {
                    alias: "next-button",
                    src: "https://raw.githubusercontent.com/user/project/refs/heads/main/assets/next-button.png",
                },
            ],
        },
    ],
};
```

</Accordion>
