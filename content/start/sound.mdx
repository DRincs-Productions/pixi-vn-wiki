The sound module is a wrapper around the [PixiJS Sound](https://github.com/pixijs/sound) library. It provides a simple interface for playing sounds and saves the current state at each <DynamicLink href="/start/labels">`step`</DynamicLink>.

È composto dai seguenti elementi di cui faremo riferimenti successivamenete:

* `sound` (manager) è il "gestore" dell'intero impianto sonoro. Viene usato per gestire i `channels`, avviare e gestire `media` musicali e ottenere info e modificare `assets` sonori. Puoi impostare dei valori per modificare l'intero livello audio, I settaggi non saranno memorizzati nei salvataggi, i suoi settaggi sono pensati soprattutto per le impostazioni del gioco.
* `channels` è un sotto livello rispetto ad sound (manager), può essere visto come un contenitore. Viene usato per contenere, avviare e gestire `media` musicali. Puoi impostare dei valori per modificare solo i media musicali contenuti. I settaggi non saranno memorizzati nei salvataggi, i suoi settaggi sono pensati soprattutto per le impostazioni del gioco.
* `media` musicali sono le istanze di un suono ché è stato avviato. Puoi impostare dei valori che verranno sono applicato a se stesso. Tutti i settaggi fatti verranno salvato nell salvataggio di gioco.  i suoi settaggi sono pensati soprattutto per essere modificato durante la sessione di gioco. Sono contenuti da un `channel`, in relazioni 1 a molti. ogni media dispone di un `alias` univoco.
* `assets` (sonori). come dice la parola sono gli asset usati durante il gioco per avviare suoni. Puoi settare dei suoi valori per influenzare tutti i media musicali che stanno usando questo asset. I settaggi non verranno salvati, sono pensati per essere modificato durante l'avvio del gioco durante il caricamento degli asset.

## Initialize

Prima di tutto vanno definiti gli <DynamicLink href="/start/assets-management#initialize-the-asset-matrix-at-project-start">asset matrix</DynamicLink> esattamente come per le altre cose. dopo di chè è condigliato caricare i suoni all'avvio del gioco con le funzioni fornoite da `sound`.

```ts tab="utils/assets-utility.ts"
import { Assets, sound } from "@drincs/pixi-vn";
import manifest from "../assets/manifest";

/**
 * Define all the assets that will be used in the game.
 * This function will be called before the game starts.
 * You can read more about assets management in the documentation: https://pixi-vn.web.app/start/assets-management.html
 */
export async function defineAssets() {
    await Assets.init({ manifest });

    // The audio bundle will be loaded in the background, so it will be available when needed, but it won't block the game start. // [!code focus]
    sound.backgroundLoadBundle("audio"); // [!code focus]
}
```

```ts tab="assets/manifest.ts"
import { AssetsManifest } from "@drincs/pixi-vn";

/**
 * Manifest for the assets used in the game.
 * You can read more about the manifest here: https://pixijs.com/8.x/guides/components/assets#loading-multiple-assets
 */
const manifest: AssetsManifest = {
    bundles: [
        {
            name: "audio",
            assets: [
                {
                    alias: "bgm_cheerful",
                    src: "https://raw.githubusercontent.com/DRincs-Productions/pixi-vn-bucket/refs/heads/main/audio/bgm_cheerful.wav",
                },
                {
                    alias: "sfx_whoosh",
                    src: "https://raw.githubusercontent.com/DRincs-Productions/pixi-vn-bucket/refs/heads/main/audio/sfx_whoosh.wav",
                },
            ],
        },
    ],
};
export default manifest;
```

Inoltre è consigliato definire i `channels` subito dopo la inizializzazione del gioco. Durante la defizizione dei `channels` è possibile settare i loro settaggi, soo tutti intuitivi e legati a PixiJS Sound ad eccezione di:

`background` (Optional): è un booleano, puoi usarlo per definire i `channels` pensati per souni di background come musiche o suoni ambientali. A differenza degli altri suoni non verranno stoppati da uno `step` narrativo all'altro.

```ts tab="main.ts"
import { Game, sound } from "@drincs/pixi-vn";

Game.init(body, {
    // ...
}).then(() => {
    sound.addChannel("bgm", { background: true });
    sound.addChannel("sfx");
    sound.defaultChannelAlias = "sfx";
});
```

## Play

<Callout type="warn">

Se non verrà spedificato alcun `channel` durante la funzione `play`, il `media` verra inserito in `sound.defaultChannelAlias`.

</Callout>

La funzionalità base è quella di avviare un suono (`asset`). Esso è possibile farlo sfruttando la funzionalità `play`. La funzione `play` genera un `media` con il quale è possibile interagire con il brano prodotto.

This function has the following parameters:

* `alias`: The alias to identify the `media`.
* `soundUrl` (Optional): The URL or path. If you have initialized the <DynamicLink href="/start/assets-management#initialize-the-asset-matrix-at-project-start">asset matrix</DynamicLink>, you can use the alias of the sound. If you don't provide the URL, then the alias is used as the URL.
* `options` (Optional): The options for the `media`.

<Accordions>

<Accordion title="sound_channel" id="channel-play">

```ts
import { sound } from "@drincs/pixi-vn";

await sound.findChannel("sfx").play("sfx_whoosh", { delay: 0.1 });
await sound.findChannel("bgm").play("bgm", "bgm_cheerful", { loop: true });
```

</Accordion>

<Accordion title="sound_manager" id="sound-manager-play">

```ts
import { sound } from "@drincs/pixi-vn";

await sound.play("sfx_whoosh", { delay: 0.1 });
await sound.play("bgm", "bgm_cheerful", { loop: true, channel: "bgm" });
```

</Accordion>

</Accordions>

## Pause and resume

```ts
const bgmMedia = sound.find("bgm");
if (bgmMedia) bgmMedia.paused = true;
// ...
if (bgmMedia) bgmMedia.paused = false;
```
