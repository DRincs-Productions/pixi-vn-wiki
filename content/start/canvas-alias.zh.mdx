凡是添加到画布的组件，都必须分配一个别名（alias）。 每个添加到画布中的组件都必须分配一个别名（`alias`）。别名（`alias`）是通过唯一字符串引用组件的方式。

<Accordions>

<Accordion title="PixiJS 中的别名是什么？" id="what-is-an-alias">

别名（`alias`）对应于 `PixiJSComponent.label`，因此请勿更改 `label`，而应使用 <DynamicLink href="/start/canvas-functions">Pixi’VN 提供的方法</DynamicLink> 来更改别名（`alias`）。

</Accordion>

</Accordions>

如果通过分配一个已存在的别名（`alias`）来添加组件，新组件将替换旧组件。

## 继承因子

如果使用已存在的别名（`alias`）添加组件，新组件除了替换旧组件外，还将继承旧组件的属性、`zIndex` 以及旧组件的 <DynamicLink href="/start/canvas-tickers">tickers</DynamicLink>。

## 编辑

要编辑画布组件的别名（`alias`），可以使用 [`canvas.editAlias`](/jsdoc/pixi-vn/index/interfaces/CanvasManagerInterface#editalias)。 如果该别名（`alias`）关联了一个或多个 <DynamicLink href="/start/canvas-tickers">tickers</DynamicLink>，它将在 ticker 中自动更新。

```ts
import { canvas } from "@drincs/pixi-vn";

canvas.editAlias("sprite1", "sprite2");
```

## 其他功能

<Accordions>

<Accordion title="游戏图层别名" id="game-layer-alias">

在 Pixi’VN 中，游戏图层是一个特殊组件，代表渲染所有游戏元素的主游戏区域。

该组件被分配了一个特殊的别名（`alias`）：[`CANVAS_APP_GAME_LAYER_ALIAS`](/jsdoc/pixi-vn/index/variables/CANVAS_APP_GAME_LAYER_ALIAS)，用于在脚本中引用游戏图层。

如果您想对整个图层运行某些 <DynamicLink href="/start/canvas-animations-effects">动画或效果</DynamicLink>，这非常有用。 某些功能在此项上不被允许，例如删除操作。

```ts
import { CANVAS_APP_GAME_LAYER_ALIAS, shakeEffect } from "@drincs/pixi-vn";

shakeEffect(CANVAS_APP_GAME_LAYER_ALIAS);
```

</Accordion>
