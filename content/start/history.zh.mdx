:::tip[术语说明]

In this documentation, the term "narrative history" refers to the list of all dialogues, choices, and more that have been shown to the player.

:::

Pixi’VN 在游戏过程中每次执行 叙事步骤（`step`）时，都会保存所有对话、选择、回应等内容。

## 获取

To get the narrative history, use [`stepHistory.narrativeHistory`](/jsdoc/pixi-vn/index/interfaces/HistoryManagerInterface#narrativehistory).

```ts
const dialogues: NarrativeHistory[] = stepHistory.narrativeHistory;
```

## 移除

To delete all narrative history, use [`stepHistory.removeNarrativeHistory`](/jsdoc/pixi-vn/index/interfaces/HistoryManagerInterface#removenarrativehistory).

```ts
stepHistory.removeNarrativeHistory();
```

如果要删除部分历史记录，传入一个数字便可移除前N条记录：

```ts
// Delete the first 2 elements
stepHistory.removeNarrativeHistory(2);
```

## 其他功能

<Accordions>

<Accordion title="Limit of saved steps" id="limit-saved-steps">

每进行一个步骤（`step`），系统都会保存当前的完整状态信息。

为了防止存档文件过大，系统对保存的步骤（`step`）数量设有限制。 默认情况下，只保存最近的20个步骤（`step`），但你可以增加这个限制（例如设为 100）。 达到限制后，只会保留较早步骤（`step`）中的必要信息。 这样你依然可以显示完整的<DynamicLink href="/start/history">历史记录</DynamicLink>，但无法<DynamicLink href="/start/labels-flow#go-back">返回</DynamicLink>到超出限制的特定步骤（`step`）。

You can change the `step` save limit by setting the [`stepHistory.stepLimitSaved`](/jsdoc/pixi-vn/index/interfaces/HistoryManagerInterface#steplimitsaved) property.

```ts
import { stepHistory } from "@drincs/pixi-vn";

stepHistory.stepLimitSaved = 100;
```

To disable the `step` save limit, set [`stepHistory.stepLimitSaved`](/jsdoc/pixi-vn/index/interfaces/HistoryManagerInterface#steplimitsaved) to `Infinity`.

```ts
import { stepHistory } from "@drincs/pixi-vn";

stepHistory.stepLimitSaved = Infinity;
```
