title: 历史记录
description: 描述 Pixi’VN 如何追踪和管理叙事历史，包括对话、选择、叙事步骤计数器和保存限制。
---

:::tip[UI 界面]

您可以在 <DynamicLink href="/start/interface-examples#history">界面示例</DynamicLink> 部分找到历史记录 UI 界面的示例。

:::

:::tip[术语说明]

在本文档中，"叙事历史"一词指的是已向玩家展示的所有对话、选择等内容的列表。

:::

Pixi’VN 在游戏执行的每个叙事步骤（`step`）中保存所有对话、选择、响应等信息。 这使得回退到之前的叙事步骤成为可能，让玩家能够回顾过去的事件或重温早期的选择。

## 获取

要获取叙事历史，请使用 [`stepHistory.narrativeHistory`](/jsdoc/pixi-vn/index/interfaces/HistoryManagerInterface#narrativehistory)。

```ts
const dialogues: NarrativeHistory[] = stepHistory.narrativeHistory;
```

## 移除

要删除所有叙事历史，请使用 [`stepHistory.removeNarrativeHistory`](/jsdoc/pixi-vn/index/interfaces/HistoryManagerInterface#removenarrativehistory)。

```ts
stepHistory.removeNarrativeHistory();
```

要删除部分叙事历史，请传入一个数字以删除前 N 个元素：

```ts
// Delete the first 2 elements
stepHistory.removeNarrativeHistory(2);
```

## 其他功能

<Accordions>

<Accordion title="保存叙事步骤数量限制（`step`）" id="limit-saved-steps">

在每个叙事步骤（`step`）执行时，当前游戏状态的所有信息都会被保存。

为防止存档文件过大，已保存的叙事步骤（`steps`）数量存在限制。 默认情况下，仅保存最近 20 个叙事步骤（`steps`），但您可以提高此限制（例如，调整为 100）。 当达到限制时，仅保留较旧叙事步骤（`steps`）的基本信息。 这允许您显示完整的 <DynamicLink href="/start/history">叙事历史</DynamicLink>，但无法 <DynamicLink href="/start/labels-flow#go-back">返回</DynamicLink> 到超出限制的特定叙事步骤（`step`）。

您可以通过设置 [`stepHistory.stepLimitSaved`](/jsdoc/pixi-vn/index/interfaces/HistoryManagerInterface#steplimitsaved) 属性来更改叙事步骤（`step`）的保存限制。

```ts
import { stepHistory } from "@drincs/pixi-vn";

stepHistory.stepLimitSaved = 100;
```

要禁用叙事步骤（`step`）保存限制，请将 [`stepHistory.stepLimitSaved`](/jsdoc/pixi-vn/index/interfaces/HistoryManagerInterface#steplimitsaved) 设置为 `Infinity`。

```ts
import { stepHistory } from "@drincs/pixi-vn";

stepHistory.stepLimitSaved = Infinity;
```
