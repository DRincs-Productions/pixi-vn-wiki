Pixi’VN позволяет разработчикам анимировать компоненты canvas с помощью функции под названием [`canvas.animate`](/jsdoc/pixi-vn/index/interfaces/CanvasManagerInterface#animate). Эта функция является переработанной реализацией [функции `animate` из библиотеки `motion`](https://motion.dev/docs/animate), адаптированной для использования тикеров PixiJS для запуска событий анимации.

<Accordions>

<Accordion title="Что такое motion?" id="what-is-motion">

`motion` — это популярная библиотека JavaScript, которая предоставляет простой и мощный способ создания анимаций. Подробнее об этом можно узнать [здесь](https://motion.dev/).

</Accordion>

</Accordions>

## Использование

[`canvas.animate`](/jsdoc/pixi-vn/index/interfaces/CanvasManagerInterface#animate) — одна из наиболее часто используемых функций в Pixi’VN, разработанная так, чтобы быть простой в использовании, хотя лежащая в основе концепция может быть не сразу очевидна. Идея проста: вы задаёте **целевое состояние** для компонента canvas (например, позицию, масштаб или прозрачность) и **длительность**, и Pixi’VN плавно, кадр за кадром, переведёт компонент в это состояние. Как только цель достигнута, анимация автоматически завершается.

Например, чтобы переместить компонент в позицию `{ x: 100, y: 50 }` за одну секунду, вы передаёте `{ x: 100, y: 50 }` в качестве [`keyframes`](/jsdoc/pixi-vn/index/interfaces/CanvasManagerInterface#keyframes) и `{ duration: 1 }` в качестве [`options`](/jsdoc/pixi-vn/index/interfaces/CanvasManagerInterface#options). Параметр [`keyframes`](/jsdoc/pixi-vn/index/interfaces/CanvasManagerInterface#keyframes) определяет, *чего* нужно достичь; [`options`](/jsdoc/pixi-vn/index/interfaces/CanvasManagerInterface#options) определяет, *как* этого достичь.

:::warning

[`canvas.animate`](/jsdoc/pixi-vn/index/interfaces/CanvasManagerInterface#animate) внутренне создаёт Ticker. Для более сложных сценариев использования — таких как приостановка, возобновление или остановка анимации — вы можете использовать <DynamicLink href="/start/canvas-tickers-functions">функции для управления Ticker'ами Pixi’VN</DynamicLink>.

:::

Вот несколько примеров:

<Accordion title="Перемещение" id="move">

```ts tab="content/labels/start.label.ts"
import { canvas, ImageSprite, newLabel, showImage } from "@drincs/pixi-vn";

export const startLabel = newLabel("start", [
    async () => {
        const alien = await showImage("alien");
        canvas.animate(alien, { xAlign: 1, yAlign: 0 }, { ease: "easeOut" });
    },
    () =>
        canvas.animate<ImageSprite>(
            "alien",
            { xAlign: 1, yAlign: 1 },
            { ease: "backOut" },
        ),
    () =>
        canvas.animate<ImageSprite>(
            "alien",
            { xAlign: 0, yAlign: 1 },
            { ease: "circIn" },
        ),
    () =>
        canvas.animate<ImageSprite>(
            "alien",
            { xAlign: 0, yAlign: 0 },
            { ease: "linear" },
        ),
]);
```

## Другие возможности

<Accordion title="Последовательность" id="sequence">

Функцию [`canvas.animate`](/jsdoc/pixi-vn/index/interfaces/CanvasManagerInterface#animate) также можно использовать для создания последовательностей анимаций.

Чтобы создать последовательность, вы можете передавать массивы в качестве значений свойств в объекте keyframes. В этом случае вы можете использовать [свойство `times`](https://motion.dev/docs/animate#times), чтобы указать тайминг каждого keyframe.

Например:

```ts tab="main.ts"
import { canvas, showImage } from "@drincs/pixi-vn";

const alien = await showImage("alien");
canvas.animate(
    alien,
    {
        xAlign: [0, 1, 1, 0, 0],
        yAlign: [0, 0, 1, 1, 0],
    },
    { repeat: Infinity, duration: 10 },
);
```

```ts tab="assets/index.ts"
import { AssetsManifest } from "@drincs/pixi-vn";

/**
 * Manifest for the assets used in the game.
 * You can read more about the manifest here: https://pixijs.com/8.x/guides/components/assets#loading-multiple-assets
 */
export const manifest: AssetsManifest = {
    bundles: [
        {
            name: "start",
            assets: [
                {
                    alias: "alien",
                    src: "https://pixijs.com/assets/eggHead.png",
                },
            ],
        },
    ],
};
```

<Accordion title="Последовательности timeline" id="timeline-sequences">

:::warning

У этого метода есть некоторые ограничения по сравнению с предыдущим, например ограничения на свойство [`repeat`](https://motion.dev/docs/animate#repeat) из-за особенностей оригинальной библиотеки `motion`.

:::

Другой способ создания последовательностей анимаций с помощью [`canvas.animate`](/jsdoc/pixi-vn/index/interfaces/CanvasManagerInterface#animate) — использование [timeline](https://motion.dev/docs/animate#timeline-sequences). Это полезно, когда вы хотите объединить в цепочку несколько анимаций, которые не являются строго линейными. Вы можете предоставить массив keyframe'ов, где каждый keyframe — это объект со свойствами для анимации и их значениями, а также необязательными параметрами анимации.

Например:

```ts tab="main.ts"
import { canvas, showImage } from "@drincs/pixi-vn";

const alien = await showImage("alien");
canvas.animate(
    alien,
    [
        [{ xAlign: 0, yAlign: 0 }, { ease: "circInOut" }],
        [{ xAlign: 1, yAlign: 0 }, { ease: "backInOut" }],
        [{ xAlign: 1, yAlign: 1 }, { ease: "linear" }],
        [{ xAlign: 0, yAlign: 1 }, { ease: "anticipate" }],
        [{ xAlign: 0, yAlign: 0 }, { ease: "easeOut" }],
    ],
    { repeat: 10, duration: 10 },
);
```

<Accordion title="Анимация PixiJS UI" id="animate-pixijs-ui">

Pixi’VN также напрямую предоставляет функцию `animate` из `motion`, которую можно импортировать так:

```ts
import { animate } from "@drincs/pixi-vn/motion";
```

В отличие от `canvas.animate`, эта функция **не** сохраняет текущее состояние анимации. Это делает её более производительной, и она является правильным выбором для анимации компонентов <DynamicLink href="/start/interface-pixijs">PixiJS UI</DynamicLink> — поскольку элементы PixiJS UI в любом случае не являются частью сохранённого состояния Pixi’VN.
