<Accordion title="Qu'est-ce que Markdown ?" id="what-is-markdown">

**Markdown** est un langage de balisage léger permettant de créer du texte formaté à l'aide d'un éditeur de texte brut. John Gruber a créé Markdown en 2004, en collaboration avec Aaron Swartz, en tant que langage de balisage conçu pour être facile à lire sous sa forme de code source. Markdown est largement utilisé pour les blogs et la messagerie instantanée, ainsi que dans les forums en ligne, les logiciels collaboratifs, les pages de documentation et les fichiers readme.

</Accordion>

</Accordions>

Pixi’VN n'est lié à aucun balisage et donne au développeur la possibilité de choisir le balisage qu'il préfère. Cependant, il est recommandé d'utiliser Markdown.

:::tip[Modèles]

Dans tous les modèles, le composant Markdown est déjà défini.

:::

:::tip[*ink*]

Vous pouvez utiliser cette méthode avec la syntaxe *ink*. Voir plus <DynamicLink href="/ink/markup#markdown">ici</DynamicLink>.

:::

Voici quelques exemples d'implémentations de Markdown dans l'écosystème JavaScript :

```tsx tab="React"
// I use the react-markdown library to convert the Markdown to HTML
// read more about it here: https://www.npmjs.com/package/react-markdown
import Markdown from "react-markdown";
import rehypeRaw from "rehype-raw";
import remarkGfm from "remark-gfm";

export default function MarkdownComponent({ text }: { text: string }) {
    return (
        <Markdown remarkPlugins={[remarkGfm]} rehypePlugins={[rehypeRaw]}>
            {text}
        </Markdown>
    );
}
```

```vue tab="Vue"
<!-- I use the vue-markdown-render library to convert the Markdown to HTML -->
<!-- read more about it here: https://www.npmjs.com/package/vue-markdown-render -->
<template>
    <div>
        <vue-markdown :source="src" />
    </div>
</template>

<script lang="ts">
import VueMarkdown from 'vue-markdown-render'

export default defineComponent({
  name: 'MyComponent',
  components: {
    VueMarkdown
  },
  setup(props, ctx) {
    const src = ref('---
title: header')
---

    return {
      src
    }
  }
})
</script>
```

```svelte tab="Svelte"
<!-- I use the svelte-markdown library to convert the Markdown to HTML -->
<!-- read more about it here: https://www.npmjs.com/package/svelte-markdown -->
<script>
  import SvelteMarkdown from 'svelte-markdown'
  const source = `
  ---
title: This is a header
---

This is a paragraph.

* This is a list
* With two items
  1. And a sublist
  2. That is ordered
    * With another
    * Sublist inside

| And this is | A table |
|-------------|---------|
| With two    | columns |`
</script>

<SvelteMarkdown {source} />
```

```tsx tab="Angular"
// I use the ngx-markdown library to convert the Markdown to HTML
// read more about it here: https://www.npmjs.com/package/ngx-markdown
import { Component, Input } from "@angular/core";
import { MarkdownModule } from "ngx-markdown";

@Component({
    selector: "app-markdown",
    template: ` <markdown [data]="text"></markdown> `,
})
export class MarkdownComponent {
    @Input() text: string;
}
```

## React Markdown Typewriter

[React Markdown Typewriter](https://www.npmjs.com/package/react-markdown-typewriter) est une bibliothèque qui combine Markdown et l'effet machine à écrire. Cette bibliothèque a été créée par moi pour répondre à mon besoin d'ajouter un effet machine à écrire au composant Markdown dans mes modèles React.

Si vous utilisez React, je vous recommande de l'utiliser :

```tsx title="React"
import { MarkdownTypewriter } from "react-markdown-typewriter";
import rehypeRaw from "rehype-raw";
import remarkGfm from "remark-gfm";

export default function MarkdownComponent({ text }: { text: string }) {
    return (
        <MarkdownTypewriter
            remarkPlugins={[remarkGfm]}
            rehypePlugins={[rehypeRaw]}
        >
            {text}
        </MarkdownTypewriter>
    );
}
```
