在视觉小说及同类游戏中，让玩家自由选择母语是非常常见的需求。

对于Pixi'VN项目，我们建议使用外部库来管理翻译内容。 这样你就可以灵活选用最契合项目需求的实现方案。

目前生态中最流行、兼容性最好的方案就是**i18next**。

![i18next-h2](/i18next.svg)

## i18next

**什么是 i18next**？i18next是一个面向JavaScript的国际化框架。 使用前，需要先安装并初始化它。 具体可查阅[i18next官网](https://www.i18next.com/)。

:::tip[模板]

所有模板均已预装并配置好i18next。 你可以直接开始翻译游戏内容。

:::

翻译内容以多个JSON文件存储（每种语言一个），采用键 - 值对形式。 键是文本的唯一标识符，值是对应语言的翻译结果。

```ts title="lib/i18n.ts"
import i18n, { type ReadCallback } from "i18next";
import LanguageDetector from "i18next-browser-languagedetector";
import ChainedBackend from "i18next-chained-backend";
import { initReactI18next } from "react-i18next";

class Backend extends ChainedBackend {
    override async read(lng: string, ns: string, callback: ReadCallback) {
        try {
            const object = await getLocalesResource(lng);
            const result = object[ns];
            callback(null, result);
        } catch (error) {
            callback(error as Error, { data: null });
        }
    }
}

export const useI18n = async () => {
    if (!i18n.isInitialized) {
        return await i18n
            .use(Backend)
            .use(LanguageDetector)
            .use(initReactI18next)
            .init({
                debug: false,
                fallbackLng: "en",
                supportedLngs: ["en"],
                interpolation: {
                    escapeValue: false,
                },
                load: "currentOnly",
                detection: {
                    order: ["localStorage"],
                    caches: ["localStorage"],
                },
                missingInterpolationHandler(_text, value, _options) {
                    return value[1];
                },
            });
    }
};

function getLocalesResource(
    lng: string,
): Promise<Record<string, Record<string, string>>> {
    return import(`./../locales/${lng}.json`);
}
```

<Accordion title="Translate the UI" id="translate-ui">

界面文本翻译只需调用 i18next 提供的`t`函数， 传入对应的翻译键，即可获取当前语言下的文本。

For example:

```tsx tab="React"
import { useTranslation } from "react-i18next";

export default function MyComponent() {
    const { t } = useTranslation("ui");

    return (
        <div>
            <Button>{t("text_speed")}</Button>
        </div>
    );
}
```

```vue tab="Vue"
<script setup>
import { useTranslation } from "i18next-vue";
const { t } = useTranslation("ui");
</script>

<template>
    <div>
        <button>{{ t("text_speed") }}</button>
    </div>
</template>
```

```svelte tab="Svelte"
<script>
    import { useTranslation } from 'svelte-i18n';
    const { t } = useTranslation("ui");
</script>

<div>
    <button>{t('text_speed')}</button>
</div>
```

:::tip

建议在<DynamicLink href="/start/labels">标签</DynamicLink>内部使用`t`函数，而不是直接在UI组件中调用。这样可以充分利用[i18n的插值功能](https://i18next.com/translation-function/interpolation)。

在Pixi'VN项目中，你经常需要在TypeScript/JavaScript<DynamicLink href="/start/labels">标签</DynamicLink>中翻译剧情文本。 为此，翻译函数必须包含在<DynamicLink href="/start/labels#steplabelprops">`StepLabelProps`接口</DynamicLink>.中。

建议直接使用原始字符串作为翻译键，便于管理与识别。

```ts tab="content/labels/start.label.ts"
export const startLabel = newLabel("start", [
    ({ t }) => {
        narration.dialogue = {
            character: liam,
            text: t("Hello, my name is {{name}}", { name: "Liam" }),
        };
    },
]);
```

```ts tab="pixi-vn.d.ts"
import type { TFunction } from "i18next";

declare module "@drincs/pixi-vn" {
    interface StepLabelProps {
        /**
         * Translate a key to a string.
         * @param key The key to translate.
         * @returns The translated string.
         */
        t: TFunction<[string], undefined>;
        // ...
    }
}
```

```ts tab="locales/es.json"
{
    "narration": {
        "Hello, my name is {{name}}": "Hola, mi nombre es {{name}}"
        // ...
    }
}
```

```ts tab="lib/hooks/props-hooks.ts"
import { useTranslation } from "react-i18next";

export function useGameProps(): StepLabelProps {
    const { t } = useTranslation(["narration"]);

    return {
        // ...

    };
}
```

## 创建存档

:::tip[*ink*]

If you use *ink*, you can automatically extract strings from *ink* text for translation. 具体方法见<DynamicLink href="/ink/translate">此</DynamicLink>。

:::

翻译用的JSON文件需手动创建。 It is recommended to split translations into two sections (see `es.json`):

* UI texts: for screens, settings, quick buttons, etc.—everything not part of the narration.
* Narration texts: for dialogues, choice menus, etc.

```json tab="locales/es.json"
{
    "ui": {
        "text_speed": "Velocidad del texto"
        // ...
    },
    "narration": {
        "Hello, my name is {{name}}": "Hola, mi nombre es {{name}}"
        // ...
    }
}
```
