---
title: Chargement des ressources
description: Apprenez à charger et gérer les ressources dans Pixi’VN en utilisant des alias, des bundles et différentes stratégies de chargement pour garder votre jeu fluide.
---

En particulier pour les jeux avec des ressources en ligne, suivre de bonnes pratiques de chargement est essentiel pour éviter de longs temps d'attente et maintenir une expérience fluide pour le joueur.

Pour charger et manipuler des ressources (images, gifs, vidéos, etc.) vous devrez utiliser `Assets`. `Assets` est une classe dotée de nombreuses fonctionnalités et provient de la bibliothèque PixiJS. Pour plus d'informations, lisez [ici](https://pixijs.com/8.x/guides/components/assets).

## Utiliser des alias pour charger des ressources

Pour charger une ressource, utilisez toujours son **alias** — l'identifiant unique défini dans le manifeste ou lors de l'enregistrement de la ressource. L'utilisation directe du chemin `src` est fortement déconseillée : elle lie votre code à l'emplacement du fichier, ce qui le rend fragile lorsque les chemins ou l'hébergement changent.

```ts title="main.ts"
import { Assets } from "@drincs/pixi-vn";

const texture = await Assets.load("eggHead");
```

```json title="assets/manifest.gen.json"
{
    "bundles": [
        {
            "name": "default",
            "assets": [
                {
                    "alias": "eggHead",
                    "src": "./assets/eggHead.png"
                }
            ]
        }
    ]
}
```

## Organiser les ressources en bundles

Une bonne pratique consiste à regrouper les ressources en **bundles** plutôt que de les enregistrer individuellement. Les bundles vous permettent de charger un ensemble complet de ressources en un seul appel, et de différer le chargement jusqu'à ce que les ressources soient réellement nécessaires.

:::tip

Pour les ressources locales, nommer les bundles en fonction de leur utilisation est moins critique — puisque les fichiers locaux se chargent instantanément, il n'y a aucun avantage réel à les séparer par écran ou `label`. Cette convention est surtout utile pour les ressources en ligne, où un chargement ciblé réduit directement les temps d'attente.

:::

Il est recommandé de nommer chaque bundle en fonction de **l'endroit où il sera utilisé**. Par exemple :

* utilisez l'identifiant du `label` (p. ex. `startLabel.id`) comme nom de bundle pour les ressources utilisées dans ce `label` spécifique
* utilisez le chemin de la route (p. ex. `"/"`) comme nom de bundle pour les ressources utilisées dans l'écran correspondant, tel que le menu principal

De cette façon, vous pouvez charger exactement les bonnes ressources au bon moment, sans charger plus que nécessaire.

```ts title="assets/index.ts"
import { startLabel } from "@/content/labels/start.label";
import type { FileRouteTypes } from "@/routeTree.gen";
import type { AssetsManifest } from "@drincs/pixi-vn";

export const manifest: AssetsManifest = {
    bundles: [
        // screens
        {
            // main menu
            name: "/" as FileRouteTypes["fullPaths"],
            assets: [
                {
                    alias: "background_main_menu",
                    src: "https://raw.githubusercontent.com/user/project/refs/heads/main/main-menu.png",
                },
            ],
        },
        // labels
        {
            name: startLabel.id,
            assets: [
                {
                    alias: "bg01-hallway",
                    src: "https://raw.githubusercontent.com/user/project/refs/heads/main/breakdown/bg01-hallway.webp",
                },
            ],
        },
    ],
};
```

## Quand charger les ressources

Par défaut, les ressources sont chargées à la demande — uniquement lorsqu'elles sont réellement nécessaires. Cependant, charger au dernier moment peut provoquer des pauses perceptibles pendant le jeu.

Voici les moments les plus courants et recommandés pour charger les ressources à l'avance :

<Accordions>

<Accordion title="Charger les ressources au démarrage du projet" id="load-assets-at-project-start">

Il est possible de charger des ressources au démarrage du projet avant que le joueur puisse interagir avec celui-ci, par exemple pendant l'écran de chargement initial. Il est conseillé de n'utiliser cette procédure que pour les ressources utilisées sur la page principale ou pour les ressources fréquemment utilisées, et de ne pas dépasser 100 Mo.

Pour ce faire, vous devez utiliser la fonction `Assets.load` et attendre qu'elle se termine (avec `await`) au démarrage du projet.

```ts tab="lib/utils/assets-utility.ts"
import { manifest } from "@/assets";
import { Assets, sound } from "@drincs/pixi-vn";

let assetsInitialized = false;

/**
 * Define all the assets that will be used in the game.
 * This function will be called before the game starts.
 */
export async function defineAssets() {
    if (!assetsInitialized) {
        await Assets.init({ manifest });
        assetsInitialized = true;
    }

    // The game will not start until these asserts are loaded.
    await Assets.loadBundle("base");
}
```

```ts tab="assets/index.ts"
import type { AssetsManifest } from "@drincs/pixi-vn";

export const manifest: AssetsManifest = {
    bundles: [
        {
            name: "base",
            assets: [
                {
                    alias: "eggHead",
                    src: "https://pixijs.com/assets/eggHead.png",
                },
                {
                    alias: "flowerTop",
                    src: "https://pixijs.com/assets/flowerTop.png",
                },
            ],
        },
    ],
};
```

</Accordion>

<Accordion title="Charger les ressources en arrière-plan au démarrage du projet" id="load-assets-in-the-background-at-project-start">

Il est possible de charger des ressources en arrière-plan au démarrage du projet, réduisant ainsi les temps de chargement au lancement. Il est recommandé de ne pas dépasser 2 Go.

Pour ce faire, vous pouvez utiliser la fonction `Assets.backgroundLoad` au démarrage du projet. Vous pouvez en savoir plus sur la fonction `Assets.backgroundLoad` [ici](https://pixijs.com/8.x/guides/components/assets).

```ts tab="lib/utils/assets-utility.ts"
import { manifest } from "@/assets";
import { Assets, sound } from "@drincs/pixi-vn";

let assetsInitialized = false;

/**
 * Define all the assets that will be used in the game.
 * This function will be called before the game starts.
 */
export async function defineAssets() {
    if (!assetsInitialized) {
        await Assets.init({ manifest });
        assetsInitialized = true;
    }

    // The game will start immediately, but these assets will be loaded in the background.
    Assets.backgroundLoadBundle(["audio"]);
}
```

```ts tab="assets/index.ts"
import type { AssetsManifest } from "@drincs/pixi-vn";

export const manifest: AssetsManifest = {
    bundles: [
        {
            name: "audio",
            assets: [
                {
                    alias: "bgm",
                    src: "https://raw.githubusercontent.com/user/project/refs/heads/main/audio/bgm.mp3",
                },
            ],
        },
    ],
};
```

</Accordion>

<Accordion title="Charger les ressources au démarrage d'un label" id="load-assets-at-label-start">

:::tip[*ink*]

Vous pouvez utiliser cette méthode avec la syntaxe *ink*. Voir plus <DynamicLink href="/ink/assets">ici</DynamicLink>.

:::

Pour regrouper les chargements d'un `step` à l'autre d'un `label` en un seul chargement, il est possible de charger tout ou partie des ressources utilisées par le `label` avant qu'il ne démarre. Étant donné que cela ne réduit pas les temps d'attente pour le joueur, mais les regroupe en un seul chargement, il est recommandé de [les charger en arrière-plan](#load-assets-in-the-background-at-label-start).

Pour ce faire, vous devez utiliser la fonction `Assets.load` et attendre qu'elle se termine (avec `await`) au démarrage du `label`. Vous utiliserez la fonction <DynamicLink href="/start/labels-advanced#onloadinglabel">`onLoadingLabel`</DynamicLink>.

Pour un code plus propre, il est recommandé de définir un bundle dans votre manifeste avec l'identifiant correspondant à l'identifiant du `label` et d'y définir les ressources qui seront utilisées dans ce `label`.

```ts tab="content/labels/start.label.ts"
import { newLabel, showImage, Assets } from "@drincs/pixi-vn";

const startLabel = newLabel(
    "start",
    [
        () => {
            await showImage("eggHead");
        },
        () => {
            await showImage("flowerTop");
        },
    ],
    {
        onLoadingLabel: async (_stepId, label) => {
            // The label will not start until these assets are loaded.
            await Assets.loadBundle(label.id);
    },
);
export default startLabel;
```

```ts tab="assets/index.ts"
import type { AssetsManifest } from "@drincs/pixi-vn";
import startLabel from "@/content/labels/start.label";

export const manifest: AssetsManifest = {
    bundles: [
        {
            name: startLabel.id,
            assets: [
                {
                    alias: "eggHead",
                    src: "https://pixijs.com/assets/eggHead.png",
                },
                {
                    alias: "flowerTop",
                    src: "https://pixijs.com/assets/flowerTop.png",
                },
            ],
        },
    ],
};
```

<Accordion title="Charger les ressources en arrière-plan au démarrage d'un label" id="load-assets-in-the-background-at-label-start">

:::tip[Modèles]

Dans tous les templates, lorsqu'un `label` est démarré, le chargement en arrière-plan du bundle (s'il existe) avec l'identifiant correspondant au `label` est lancé. Il suffit donc de **créer un manifeste avec un bundle pour chaque `label`**.

:::

Pour rendre le jeu plus fluide en essayant de supprimer les temps de chargement des ressources d'un `step` à l'autre, il est possible de démarrer un "groupe de chargement" au début d'un `label`. Cela signifie que vous ne ressentirez potentiellement aucun chargement, notamment dans les `steps` ultérieurs du `label`.

Pour ce faire, vous pouvez utiliser la fonction `Assets.backgroundLoad` au démarrage du `label`. Vous pouvez en savoir plus sur la fonction `Assets.backgroundLoad` [ici](https://pixijs.com/8.x/guides/components/assets). Et vous utiliserez la fonction <DynamicLink href="/start/labels-advanced#onloadinglabel">`onLoadingLabel`</DynamicLink>.

Pour un code plus propre, il est recommandé de définir un bundle dans votre manifeste avec l'identifiant correspondant à l'identifiant du `label` et d'y définir les ressources qui seront utilisées dans ce `label`.

```ts tab="content/labels/start.label.ts"
import { newLabel, showImage, Assets } from "@drincs/pixi-vn";

const startLabel = newLabel(
    "start",
    [
        () => {
            await showImage("eggHead");
        },
        () => {
            await showImage("flowerTop");
        },
    ],
    {
        onLoadingLabel: async (_stepId, label) => {
            // The label will start immediately, but these assets will be loaded in the background.
            Assets.backgroundLoadBundle(label.id);
        },
    },
);
export default startLabel;
```

```ts tab="assets/index.ts"
import type { AssetsManifest } from "@drincs/pixi-vn";

export const manifest: AssetsManifest = {
    bundles: [
        {
            name: startLabel.id,
            assets: [

                    alias: "eggHead",

                },
                {
                    alias: "flowerTop",
                    src: "https://pixijs.com/assets/flowerTop.png",
                },
            ],
        },
    ],
};
```

<Accordion title="Charger les ressources avant l'affichage d'un écran" id="load-assets-before-screen">

Il est possible de charger des ressources avant qu'un écran ne soit affiché. Cela est utile lorsqu'un écran utilise des images ou d'autres ressources qui doivent être prêtes avant le rendu du composant.

Dans cet exemple, TanStack Router est utilisé, qui est déjà inclus dans tous les templates. Vous pouvez trouver plus d'informations à ce sujet <DynamicLink href="/start/interface-navigate">ici</DynamicLink>.

La fonction `loader` s'exécute avant le montage du composant de route. Vous pouvez appeler `Assets.loadBundle` dans `loader` pour précharger le bundle associé à cette route.

De plus, vous pouvez utiliser `pendingComponent` pour afficher un indicateur de chargement pendant que les ressources sont récupérées, afin que l'utilisateur voie une transition significative plutôt qu'un écran vide.

```tsx tab="routes/game/narration.tsx"
import { NarrationPendingComponent } from "@/components/loading";
import { NarrationScreen } from "@/components/screens/narration";
import { Assets } from "@drincs/pixi-vn";
import { createFileRoute } from "@tanstack/react-router";

export const Route = createFileRoute("/game/narration")({
    component: NarrationElement,
    pendingComponent: NarrationPendingComponent,
    loader: async () => {
        await Assets.loadBundle("/game/narration");
    },
});

function NarrationElement() {
    return <NarrationScreen />;
}
```

```ts tab="assets/index.ts"
import type { AssetsManifest } from "@drincs/pixi-vn";
import startLabel from "@/content/labels/start.label";

export const manifest: AssetsManifest = {
    bundles: [
        {
            name: "/game/narration",
            assets: [
                {
                    alias: "next-button",
                    src: "https://raw.githubusercontent.com/user/project/refs/heads/main/assets/next-button.png",
                },
            ],
        },
    ],
};
```

</Accordion>
