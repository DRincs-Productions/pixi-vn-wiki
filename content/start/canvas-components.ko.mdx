Pixi’VN은 캔버스 컴포넌트 세트를 제공합니다. 이 컴포넌트들은 [Pixi.js](https://pixijs.com/) 컴포넌트를 확장하여 현재 상태를 저장하는 기능 등 다양한 기능을 추가합니다.

**캔버스 컴포넌트란 무엇인가요?** 컴포넌트는 캔버스 앱을 위한 재사용 가능한 구성 요소로, 앱 제작자가 앱 내부 또는 컴포넌트 라이브러리를 통해 여러 앱에서 사용할 사용자 정의 컨트롤을 만들 수 있게 해줍니다. 컴포넌트는 사용자 정의 속성과 같은 고급 기능을 사용할 수 있으며 복잡한 기능을 구현할 수 있습니다.

## 기타 기능

<Accordions>

<Accordion title="사용자 정의 컴포넌트" id="custom-components">

기본 컴포넌트를 확장하여 사용자 정의 컴포넌트를 만들 수 있습니다. 이를 위해 데코레이터 [`@canvasComponentDecorator`](/jsdoc/pixi-vn/index/functions/canvasComponentDecorator)를 사용해야 합니다.

사용자 정의 컴포넌트 속성을 저장하려면 [`memory`](/jsdoc/pixi-vn/index/classes/CanvasBaseItem#memory) 게터와 [`setMemory`](/jsdoc/pixi-vn/index/classes/CanvasBaseItem#setmemory) 메서드를 반드시 재정의해야 합니다.
`get memory()`에서는 `className` 속성을 반환하는 것이 매우 중요합니다. 이 속성은 데코레이터에서 사용된 id와 동일해야 합니다.

예를 들어, `Sprite` 클래스를 확장하는 `AlienTinting` 클래스를 만들어 애니메이션에서 각 외계인의 방향과 속도를 관리할 수 있습니다.

```ts title="canvas/components/AlienTinting.ts"
const ALIEN_TINTING_TEST_ID = "AlienTintingTest";
@canvasComponentDecorator({
    name: ALIEN_TINTING_TEST_ID,
})
class AlienTintingTest extends Sprite<IAlienTintingMemory> {
    readonly pixivnId: string = CANVAS_SPINE_ID;
    override get memory() {
        return {
            ...super.memory,
            pixivnId: CANVAS_SPINE_ID,
            direction: this.direction,
            turningSpeed: this.turningSpeed,
            speed: this.speed,
        };
    }
    override async setMemory(memory: IAlienTintingMemory) {
        await super.setMemory(memory);
        this.direction = memory.direction;
        this.turningSpeed = memory.turningSpeed;
        this.speed = memory.speed;
    }
    direction: number = 0;
    turningSpeed: number = 0;
    speed: number = 0;
    static override from(
        source: Texture | TextureSourceLike,
        skipCache?: boolean,
    ) {
        let sprite = Sprite.from(source, skipCache);
        let mySprite = new AlienTintingTest();
        mySprite.texture = sprite.texture;
        return mySprite;
    }
}
```
