---
title: 资源管理
description: Organization, referencing, and loading of assets (images, sounds, videos) in Pixi’VN using aliases and manifests.
---

要加载和操作资源（图像、gif、视频等）， 需要使用 `Assets`。 `Assets` 是一个功能丰富的类，来自 PixiJS 库。 更多信息请参阅[此处](https://pixijs.com/8.x/guides/components/assets)。

在 Pixi'VN 中使用别名和清单来组织、引用和加载资源（图像、声音、视频）。

```ts
let alien1 = await showImage("alien", "https://pixijs.com/assets/eggHead.png");
```

这种方法有一些缺点：

* 从一个版本到另一个版本更改资源的 URL 可能导致不兼容。
* 每次玩家按下"继续"并且启动了一个使用资源的<DynamicLink href="/start/labels">`step`</DynamicLink>时，玩家都必须等待短暂的加载时间。
* 在代码中写入完整的 URL 会使代码变长且可读性降低。

出于这些原因，建议通过以下方式处理资源。

## 在项目启动时初始化资源矩阵

在项目开始时初始化资源矩阵，使您可以通过唯一的别名引用资源，而无需使用 URL/路径。 使用别名可以更改资源的 URL，而无需担心版本兼容性。

为此，建议创建一个异步函数 `defineAssets`，在项目启动时调用。
在 `defineAssets` 函数中，使用 `Assets`（例如 `Assets.add`、`Assets.addBundle` 和 `Assets.init`）为 URL 分配别名。 您可以[在此处](https://pixijs.com/8.x/guides/components/assets)找到有关为每个资源分配别名的更多信息。

```ts tab="utils/defineAssets.ts"
import { Assets, sound } from "@drincs/pixi-vn";
import manifest from "../assets/manifest";

export async function defineAssets() {
    // manifest
    await Assets.init({ manifest });
    // single asset
    Assets.add({
        alias: "eggHead",
        src: "https://pixijs.com/assets/eggHead.png",
    });
    Assets.add({
        alias: "flowerTop",
        src: "https://pixijs.com/assets/flowerTop.png",
    });
    Assets.add({ alias: "video", src: "https://pixijs.com/assets/video.mp4" });
    sound.add("bird", "https://pixijs.io/sound/examples/resources/bird.mp3");
    sound.add(
        "musical",
        "https://pixijs.io/sound/examples/resources/musical.mp3",
    );
    // bundle
    Assets.addBundle("liam", {
        "liam-head": "liam_head.png",
        "liam-body": "liam_body.png",
        "liam-arms": "liam_arms.png",
    });
}
```

```ts tab="assets/manifest.ts"
import { AssetsManifest } from "@drincs/pixi-vn";

/**
 * Manifest for the assets used in the game.
 * You can read more about the manifest here: https://pixijs.com/8.x/guides/components/assets#loading-multiple-assets
 */
const manifest: AssetsManifest = {
    bundles: [
        // screens
        {
            name: "main_menu",
            assets: [
                {
                    alias: "background_main_menu",
                    src: "https://raw.githubusercontent.com/DRincs-Productions/pixi-vn-bucket/refs/heads/main/main-menu.png",
                },
            ],
        },
        // labels
        {
            name: "start",
            assets: [
                {
                    alias: "bg01-hallway",
                    src: "https://raw.githubusercontent.com/DRincs-Productions/pixi-vn-bucket/refs/heads/main/breakdown/bg01-hallway.webp",
                },
            ],
        },
    ],
};
export default manifest;
```

## 加载资源

默认情况下，资源仅在需要时加载。

但是**如果资源未保存在本地**，而是托管在<DynamicLink href="/start/assets#assets-hosting">"资源托管"</DynamicLink>中，其加载可能需要一些时间。 这意味着启动一个 `step` 可能不是即时完成的。 因此，在开始执行<DynamicLink href="/start/labels-flow#continue">下一个 `step`</DynamicLink>（例如通过"继续"按钮）后，玩家可能需要等待一段时间才能"看到"变化并运行另一个 `step`。

在每个 `step` 执行这些加载可能会让玩家感到厌烦，即使它们非常短暂。

为了解决这个问题，开发者可以在游戏的某个阶段将多次加载合并为一组。 这样一来，玩家就不会遇到连续的加载，而是次数更少、但可能更长的加载。

以下是加载资源的几种方式：

<Accordions>

<Accordion title="在项目启动时加载资源" id="load-assets-at-project-start">

可以在项目启动时、玩家能够与项目交互之前加载资源，例如在启动加载画面期间。 建议仅对主页面使用的资源或频繁使用的资源采用此方法，并且不要超过 100MB。

为此，您必须在项目启动时使用 `Assets.load` 函数并等待其完成（使用 `await`）。 您可以[在此处](https://pixijs.com/8.x/guides/components/assets)了解更多关于 `Assets.load` 函数的信息。

```ts tab="utils/defineAssets.ts"
import { Assets, sound } from "@drincs/pixi-vn";
import manifest from "../assets/manifest";

export async function defineAssets() {
    // manifest
    await Assets.init({ manifest });
    // single asset
    Assets.add({
        alias: "eggHead",
        src: "https://pixijs.com/assets/eggHead.png",
    });
    Assets.add({
        alias: "flowerTop",
        src: "https://pixijs.com/assets/flowerTop.png",
    });
    Assets.add({ alias: "video", src: "https://pixijs.com/assets/video.mp4" });
    sound.add("bird", "https://pixijs.io/sound/examples/resources/bird.mp3");
    sound.add(
        "musical",
        "https://pixijs.io/sound/examples/resources/musical.mp3",
    );
    // bundle
    Assets.addBundle("liam", {
        "liam-head": "liam_head.png",
        "liam-body": "liam_body.png",
        "liam-arms": "liam_arms.png",
    });

    // The game will not start until these assets are loaded. // [!code focus]
    await Assets.load("eggHead"); // [!code focus]
    await Assets.loadBundle("liam"); // [!code focus]
}
```

<Accordion title="在项目启动时后台加载资源" id="load-assets-in-the-background-at-project-start">

可以在项目启动时在后台加载资源，从而缩短项目的启动加载时间。 建议不要超过 2GB。

为此，您可以在项目启动时使用 `Assets.backgroundLoad` 函数。 您可以[在此处](https://pixijs.com/8.x/guides/components/assets)了解更多关于 `Assets.backgroundLoad` 函数的信息。

```ts tab="utils/defineAssets.ts"
import { Assets, sound } from "@drincs/pixi-vn";
import manifest from "../assets/manifest";

export async function defineAssets() {
    // manifest
    await Assets.init({ manifest });
    // single asset
    Assets.add({
        alias: "eggHead",
        src: "https://pixijs.com/assets/eggHead.png",
    });
    Assets.add({ alias: "video", src: "https://pixijs.com/assets/video.mp4" });
    sound.add("bird", "https://pixijs.io/sound/examples/resources/bird.mp3");
    sound.add(
        "musical",
        "https://pixijs.io/sound/examples/resources/musical.mp3",
    );
    // bundle
    Assets.addBundle("liam", {
        "liam-head": "liam_head.png",
        "liam-body": "liam_body.png",
        "liam-arms": "liam_arms.png",
    });

    // The game will not start until these assets are loaded.
    await Assets.load("eggHead");

    // The game will start immediately, but these assets will be loaded in the background. // [!code focus]
    Assets.backgroundLoadBundle(["liam"]); // [!code focus]
    // Load an individual asset in the background // [!code focus]
    Assets.backgroundLoad({
        alias: "flowerTop",
        src: "https://pixijs.com/assets/flowerTop.png",
    }); // [!code focus]
}
```

<Accordion title="在标签启动时加载资源" id="load-assets-at-label-start">

<Callout title="ink" type="info">

您可以使用 *ink* 语法使用此方法。 查看更多信息请参阅<DynamicLink href="/ink/assets">此处</DynamicLink>。

</Callout>

为了将 `label` 中从一个 `step` 到另一个 `step` 的加载合并为单次加载，可以在 `label` 启动之前加载该 `label` 使用的全部或部分资源。 由于这并不会减少玩家的等待时间，而是将其合并到单次加载中，因此建议[在后台加载它们](#load-assets-in-the-background-at-label-start)。

为此，您必须在 `label` 启动时使用 `Assets.load` 函数并等待其完成（使用 `await`）。 您将使用<DynamicLink href="/start/labels-advanced#onloadinglabel">`onLoadingLabel`</DynamicLink>函数。

为了使代码更整洁，建议在清单中定义一个与 `label` id 对应的资源包，并在其中定义该 `label` 将使用的资源。

```ts title="content/labels/start.label.ts"
import { newLabel, showImage, Assets } from "@drincs/pixi-vn";

const startLabel = newLabel(
    "start",
    [
        () => {
            await showImage("eggHead");
        },
        () => {
            await showImage("flowerTop");
        },
    ],
    {
        onLoadingLabel: async (_stepId, label) => {
            // [!code focus]
            // The label will not start until these assets are loaded. // [!code focus]
            await Assets.loadBundle(label.id); // [!code focus]
        }, // [!code focus]
    },
);
export default startLabel;
```

```ts tab="assets/manifest.ts"
import { AssetsManifest } from "@drincs/pixi-vn";
import startLabel from "../labels/startLabel";

/**
 * Manifest for the assets used in the game.
 * You can read more about the manifest here: https://pixijs.com/8.x/guides/components/assets#loading-multiple-assets
 */
const manifest: AssetsManifest = {
    bundles: [
        // labels
        {
            name: startLabel.id,
            assets: [
                {
                    alias: "eggHead",
                    src: "https://pixijs.com/assets/eggHead.png",
                },
                {
                    alias: "flowerTop",
                    src: "https://pixijs.com/assets/flowerTop.png",
                },
            ],
        },
    ],
};
export default manifest;
```

<Accordion title="在标签启动时后台加载资源" id="load-assets-in-the-background-at-label-start">

<Callout title="模板中已存在" type="info">

在所有模板中，当 `label` 启动时，如果存在与 `label` 对应的 id 的资源包，将启动该资源包的后台加载。 因此，只需**为每个 `label` 创建一个带有资源包的清单**即可。 以下是用于实现此功能的代码：

```ts title="main.ts"
import { Assets, Game } from "@drincs/pixi-vn";

Game.onLoadingLabel((_stepId, { id }) => Assets.backgroundLoadBundle(id));
```

</Callout>

为了让游戏更加流畅，通过尝试消除从一个 `step` 到另一个 `step` 的资源加载等待时间，可以在 `label` 开始时启动一个"加载组"。 这意味着您可能感觉不到任何加载，尤其是在 `label` 的后续 `steps` 中。

为此，您可以在 `label` 启动时使用 `Assets.backgroundLoad` 函数。 您可以[在此处](https://pixijs.com/8.x/guides/components/assets)了解更多关于 `Assets.backgroundLoad` 函数的信息。 并且，您将使用<DynamicLink href="/start/labels-advanced#onloadinglabel">`onLoadingLabel`</DynamicLink>函数。

为了使代码更整洁，建议在清单中定义一个与 `label` id 对应的资源包，并在其中定义该 `label` 将使用的资源。

```ts title="content/labels/start.label.ts"
import { newLabel, showImage, Assets } from "@drincs/pixi-vn";

            Assets.backgroundLoadBundle(label.id); // [!code focus]
        }, // [!code focus]
    },
);
export default startLabel;
```
