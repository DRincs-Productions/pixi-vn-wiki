---

특히 온라인 에셋이 포함된 게임에서는 올바른 로딩 방법을 따르는 것이 필수적입니다. 이를 통해 긴 대기 시간을 방지하고 플레이어의 경험을 원활하게 유지할 수 있습니다.

에셋(이미지, gif, 동영상 등) 을 로드하고 조작하려면 `Assets`를 사용해야 합니다. `Assets`는 PixiJS 라이브러리에서 제공하는 다양한 기능을 갖춘 클래스입니다. 자세한 내용은 [여기](https://pixijs.com/8.x/guides/components/assets)를 참조하세요.

## 별칭을 사용하여 에셋 로드

에셋을 로드할 때는 항상 \*\*별칭(alias)\*\*을 사용하세요. 별칭은 매니페스트에서 정의하거나 에셋을 등록할 때 지정하는 고유 식별자입니다. `src` 경로를 직접 사용하는 것은 강력히 권장하지 않습니다. 파일 위치에 코드가 종속되어 경로나 호스팅이 변경될 때 취약해집니다.

```ts title="main.ts"
import { Assets } from "@drincs/pixi-vn";

const texture = await Assets.load("eggHead");
```

```json title="assets/manifest.gen.json"
{
    "bundles": [
        {
            "name": "default",
            "assets": [
                {
                    "alias": "eggHead",
                    "src": "./assets/eggHead.png"
                }
            ]
        }
    ]
}
```

## 에셋을 번들로 구성

에셋을 개별적으로 등록하는 것보다 **번들**로 묶는 것이 모범 사례입니다. 번들을 사용하면 단일 호출로 에셋 전체 세트를 로드하고, 실제로 필요한 시점까지 로딩을 지연할 수 있습니다.

:::tip

로컬 에셋의 경우, 번들 이름을 사용 위치 기반으로 짓는 것은 크게 중요하지 않습니다. 로컬 파일은 즉시 로드되므로 화면이나 내러티브 노드 (`label`)별로 분리하는 실질적인 이점이 없습니다. 이 방식은 타겟 로딩이 대기 시간을 직접적으로 줄이는 온라인 에셋에서 가장 유용합니다.

:::

각 번들의 이름은 **사용될 위치**를 기반으로 짓는 것이 권장됩니다. 예를 들어:

* 특정 내러티브 노드 (`label`)에서 사용되는 에셋의 번들 이름으로는 해당 내러티브 노드 (`label`)의 id(예: `startLabel.id`)를 사용하세요.
* 메인 메뉴와 같이 해당 화면에서 사용되는 에셋의 번들 이름으로는 라우트 경로(예: `"/"`)를 사용하세요.

이렇게 하면 필요한 순간에 정확히 필요한 에셋만 로드하여 불필요한 로딩을 줄일 수 있습니다.

```ts title="assets/index.ts"
import { startLabel } from "@/content/labels/start.label";
import type { FileRouteTypes } from "@/routeTree.gen";
import type { AssetsManifest } from "@drincs/pixi-vn";

export const manifest: AssetsManifest = {
    bundles: [
        // screens
        {
            // main menu
            name: "/" as FileRouteTypes["fullPaths"],
            assets: [
                {
                    alias: "background_main_menu",
                    src: "https://raw.githubusercontent.com/user/project/refs/heads/main/main-menu.png",
                },
            ],
        },
        // labels
        {
            name: startLabel.id,
            assets: [
                {
                    alias: "bg01-hallway",
                    src: "https://raw.githubusercontent.com/user/project/refs/heads/main/breakdown/bg01-hallway.webp",
                },
            ],
        },
    ],
};
```

## 에셋을 로드하는 시점

기본적으로 에셋은 실제로 필요할 때만 온디맨드 방식으로 로드됩니다. 그러나 마지막 순간에 로드하면 게임 플레이 중 눈에 띄는 중단이 발생할 수 있습니다.

다음은 에셋을 미리 로드하는 가장 일반적이고 권장되는 시점입니다:

<Accordions>

<Accordion title="프로젝트 시작 시 에셋 로드" id="load-assets-at-project-start">

플레이어가 프로젝트와 상호작용하기 전, 예를 들어 시작 로딩 화면 중에 프로젝트 시작 시 에셋을 로드할 수 있습니다. 이 방법은 메인 페이지에서 사용되거나 자주 사용되는 에셋에만 사용하고, 100MB를 초과하지 않도록 권장합니다.

이를 위해 프로젝트가 시작될 때 `Assets.load` 함수를 사용하고 (`await`로) 완료될 때까지 기다려야 합니다.

```ts tab="lib/utils/assets-utility.ts"
import { manifest } from "@/assets";
import { Assets, sound } from "@drincs/pixi-vn";

let assetsInitialized = false;

/**
 * Define all the assets that will be used in the game.
 * This function will be called before the game starts.
 */
export async function defineAssets() {
    if (!assetsInitialized) {
        await Assets.init({ manifest });
        assetsInitialized = true;
    }

    // The game will not start until these asserts are loaded.
    await Assets.loadBundle("base");
}
```

```ts tab="assets/index.ts"
import type { AssetsManifest } from "@drincs/pixi-vn";

export const manifest: AssetsManifest = {
    bundles: [
        {
            name: "base",
            assets: [
                {
                    alias: "eggHead",
                    src: "https://pixijs.com/assets/eggHead.png",
                },
                {
                    alias: "flowerTop",
                    src: "https://pixijs.com/assets/flowerTop.png",
                },
            ],
        },
    ],
};
```

</Accordion>

<Accordion title="프로젝트 시작 시 백그라운드에서 에셋 로드" id="load-assets-in-the-background-at-project-start">

프로젝트가 시작될 때 백그라운드에서 에셋을 로드하여 프로젝트 시작 시 로딩 시간을 줄일 수 있습니다. 2GB를 초과하지 않도록 권장합니다.

이를 위해 프로젝트가 시작될 때 `Assets.backgroundLoad` 함수를 사용할 수 있습니다. `Assets.backgroundLoad` 함수에 대한 자세한 내용은 [여기](https://pixijs.com/8.x/guides/components/assets)를 참조하세요.

```ts tab="lib/utils/assets-utility.ts"
import { manifest } from "@/assets";
import { Assets, sound } from "@drincs/pixi-vn";

let assetsInitialized = false;

/**
 * Define all the assets that will be used in the game.
 * This function will be called before the game starts.
 */
export async function defineAssets() {
    if (!assetsInitialized) {
        await Assets.init({ manifest });
        assetsInitialized = true;
    }

    // The game will start immediately, but these assets will be loaded in the background.
    Assets.backgroundLoadBundle(["audio"]);
}
```

```ts tab="assets/index.ts"
import type { AssetsManifest } from "@drincs/pixi-vn";

export const manifest: AssetsManifest = {
    bundles: [
        {
            name: "audio",
            assets: [
                {
                    alias: "bgm",
                    src: "https://raw.githubusercontent.com/user/project/refs/heads/main/audio/bgm.mp3",
                },
            ],
        },
    ],
};
```

</Accordion>

<Accordion title="내러티브 노드 시작 시 에셋 로드" id="load-assets-at-label-start">

:::tip[*ink*]

이 메서드는 *ink* 문법과 함께 사용할 수 있습니다. 자세한 내용은 <DynamicLink href="/ink/assets">여기</DynamicLink>를 참조하세요.

:::

한 내러티브 노드 (`label`)의 내러티브 단계 (`step`)에서 다음 내러티브 단계 (`step`)로의 로딩을 하나의 로딩으로 묶기 위해, 내러티브 노드 (`label`)가 시작되기 전에 해당 내러티브 노드 (`label`)에서 사용되는 에셋의 전부 또는 일부를 로드할 수 있습니다. 이는 플레이어의 대기 시간을 줄이는 것이 아니라 하나의 로딩으로 합치는 것이므로, [백그라운드에서 로드하는 것](#load-assets-in-the-background-at-label-start)을 권장합니다.

이를 위해 내러티브 노드 (`label`)가 시작될 때 `Assets.load` 함수를 사용하고 (`await`로) 완료될 때까지 기다려야 합니다. <DynamicLink href="/start/labels-advanced#onloadinglabel">`onLoadingLabel`</DynamicLink> 함수를 사용하게 됩니다.

더 깔끔한 코드를 위해, 매니페스트에서 해당 내러티브 노드 (`label`) id에 대응하는 id를 가진 번들을 정의하고, 그 안에 해당 내러티브 노드 (`label`)에서 사용될 에셋을 정의하는 것이 권장됩니다.

```ts tab="content/labels/start.label.ts"
import { newLabel, showImage, Assets } from "@drincs/pixi-vn";

const startLabel = newLabel(
    "start",
    [
        () => {
            await showImage("eggHead");
        },
        () => {
            await showImage("flowerTop");
        },
    ],
    {
        onLoadingLabel: async (_stepId, label) => {
            // The label will not start until these assets are loaded.
            await Assets.loadBundle(label.id);
    },
);
export default startLabel;
```

```ts tab="assets/index.ts"
import type { AssetsManifest } from "@drincs/pixi-vn";
import startLabel from "@/content/labels/start.label";

export const manifest: AssetsManifest = {
    bundles: [
        {
            name: startLabel.id,
            assets: [
                {
                    alias: "eggHead",
                    src: "https://pixijs.com/assets/eggHead.png",
                },
                {
                    alias: "flowerTop",
                    src: "https://pixijs.com/assets/flowerTop.png",
                },
            ],
        },
    ],
};
```

<Accordion title="내러티브 노드 시작 시 백그라운드에서 에셋 로드" id="load-assets-in-the-background-at-label-start">

:::tip[템플릿]

모든 템플릿에서 내러티브 노드 (`label`)가 시작될 때, 해당 내러티브 노드 (`label`) id에 대응하는 id를 가진 번들(존재하는 경우)의 백그라운드 로딩이 시작됩니다. 따라서 **각 내러티브 노드 (`label`)에 대한 번들을 포함하는 매니페스트를 만드는 것**으로 충분합니다.

:::

한 내러티브 단계 (`step`)에서 다음 내러티브 단계 (`step`)로의 에셋 로딩 시간을 제거하여 게임을 더욱 원활하게 만들기 위해, 내러티브 노드 (`label`) 시작 시 "로딩 그룹"을 시작할 수 있습니다. 이를 통해 특히 내러티브 노드 (`label`)의 후반 내러티브 단계 (`step`)에서 로딩을 전혀 느끼지 못할 수 있습니다.

이를 위해 내러티브 노드 (`label`)가 시작될 때 `Assets.backgroundLoad` 함수를 사용할 수 있습니다. `Assets.backgroundLoad` 함수에 대한 자세한 내용은 [여기](https://pixijs.com/8.x/guides/components/assets)를 참조하세요. 그리고 <DynamicLink href="/start/labels-advanced#onloadinglabel">`onLoadingLabel`</DynamicLink> 함수를 사용하게 됩니다.

더 깔끔한 코드를 위해, 매니페스트에서 해당 내러티브 노드 (`label`) id에 대응하는 id를 가진 번들을 정의하고, 그 안에 해당 내러티브 노드 (`label`)에서 사용될 에셋을 정의하는 것이 권장됩니다.

```ts tab="content/labels/start.label.ts"
import { newLabel, showImage, Assets } from "@drincs/pixi-vn";

const startLabel = newLabel(
    "start",
    [
        () => {
            await showImage("eggHead");
        },
        () => {
            await showImage("flowerTop");
        },
    ],
    {
        onLoadingLabel: async (_stepId, label) => {
            // The label will start immediately, but these assets will be loaded in the background.
            Assets.backgroundLoadBundle(label.id);
        },
    },
);
export default startLabel;
```

```ts tab="assets/index.ts"
import type { AssetsManifest } from "@drincs/pixi-vn";
import startLabel from "@/content/labels/start.label";

export const manifest: AssetsManifest = {
    bundles: [
        {
            name: startLabel.id,
            assets: [
                {
                    alias: "eggHead",
                    src: "https://pixijs.com/assets/eggHead.png",
                },
                {
                    alias: "flowerTop",
                    src: "https://pixijs.com/assets/flowerTop.png",
                },
            ],
        },
    ],
};
```

<Accordion title="화면 표시 전 에셋 로드" id="load-assets-before-screen">

화면이 표시되기 전에 에셋을 로드할 수 있습니다. 이는 컴포넌트가 렌더링되기 전에 이미지나 기타 리소스가 준비되어 있어야 하는 화면에 유용합니다.

이 예시에서는 모든 템플릿에 이미 포함된 TanStack Router를 사용합니다. 자세한 내용은 <DynamicLink href="/start/interface-navigate">여기</DynamicLink>를 참조하세요.

`loader` 함수는 라우트 컴포넌트가 마운트되기 전에 실행됩니다. `loader` 내에서 `Assets.loadBundle`을 호출하여 해당 라우트와 연관된 번들을 미리 로드할 수 있습니다.

또한 에셋이 가져오는 동안 로딩 인디케이터를 표시하기 위해 `pendingComponent`를 사용할 수 있습니다. 이를 통해 사용자는 빈 화면 대신 의미 있는 전환 화면을 볼 수 있습니다.

```tsx tab="routes/game/narration.tsx"
import { NarrationPendingComponent } from "@/components/loading";
import { NarrationScreen } from "@/components/screens/narration";
import { Assets } from "@drincs/pixi-vn";
import { createFileRoute } from "@tanstack/react-router";

export const Route = createFileRoute("/game/narration")({
    component: NarrationElement,
    pendingComponent: NarrationPendingComponent,
    loader: async () => {
        await Assets.loadBundle("/game/narration");
    },
});

function NarrationElement() {
    return <NarrationScreen />;
}
```

```ts tab="assets/index.ts"
import type { AssetsManifest } from "@drincs/pixi-vn";
import startLabel from "@/content/labels/start.label";

export const manifest: AssetsManifest = {
    bundles: [
        {
            name: "/game/narration",
            assets: [
                {
                    alias: "next-button",
                    src: "https://raw.githubusercontent.com/user/project/refs/heads/main/assets/next-button.png",
                },
            ],
        },
    ],
};
```
