什么是**节点（knot）**？ 正如[官方文档](https://www.inklestudios.com/ink/web-tutorial/)所解释的那样，故事由多个相互链接的部分组成，在**_ink_术语**中我们将其称之为“节点”。 在 ink 中，节点名称的左侧至少使用两个等号，右侧也可以加上等号（例如：=== london ===）。

标签（label）和节点（knot）有区别吗？ 没有，它们是同一个东西。 在 Pixi'VN 中，当你用 ink 创建一个节点时，内部将会创建一个“标签”，并将其保存在注册表中。

## 运行一个节点

要运行一个节点，你可以使用以下方法：

<Accordions>

<Accordion title="ink_jump" id="jump">

你可以使用 `->` 符号来运行一个结。 它与Pixi'VN库中的<DynamicLink href="/start/labels#jump-to-a-label"> “jump”功能</DynamicLink>相对应。

编写 -> 符号，后接节点的名称，例如：

```ink title="ink"
=== start ===
Start
-> after // [!code focus]

=== after ===
After
End
-> DONE
```

<Accordion title="ink_call" id="call">

你可以使用 `->` 符号来运行一个结。 它与Pixi'VN库中的<DynamicLink href="/start/labels#call-to-a-label">`call`功能</DynamicLink>相对应。

编写 <- 符号，后接节点的名称，例如：

```ink title="ink"
== start ==
I had a headache; threading is hard to get your head around.
<- conversation // [!code focus]
<- walking // [!code focus]


== conversation ==
It was a tense moment for Monty and me.
 * "What did you have for lunch today?"[] I asked.
    "Spam and eggs," he replied.
 * "Nice weather, we're having,"[] I said.
    "I've seen better," he replied.
 - -> house

== walking ==
We continued to walk down the dusty road.
 * [Continue walking]
    -> house

== house ==
Before long, we arrived at his house.
-> DONE
```
